# CoMAM 详解

> 论文：Collaborative Multi-Agent Optimization for Personalized Memory System (CoMAM)
> arxiv：2603.12631，2026 年 3 月
> 在本工作中的角色：**SpeakerMem-R1 v2 的训练 paradigm 借鉴 —— joint multi-agent end-to-end RL**

---

## 1. 它在解决什么问题

### 1.1 一句话

CoMAM 是第一个**端到端联合训练**记忆系统多个 agent（construction + retrieval 等）的 RL 框架，证明 **joint > sequential**。

### 1.2 它批判了谁

CoMAM 的核心批判 Memory-R1：

> Memory-R1 优化 memory construction agent 和 retrieval agent **分别**——construction 训完冻结，再训 retrieval。但这两个 agent 的目标其实是耦合的：construction 写的好坏直接决定 retrieval 能取到什么；retrieval 的需求反过来应该指导 construction 写什么。Sequential 训练错过了这个反馈。

CoMAM 把整个 pipeline 当成一个 MDP，所有 agent 用同一个 policy update step 来更新（at least 在 gradient 层面 joint）。

### 1.3 它对我们意味着什么

我们 SpeakerMem-R1 v1 设计中没明确说"joint vs sequential"，但 v2 必须**明确 joint**，否则会被 reviewer 拿 CoMAM 来砸：

> "Why don't you do joint training as in CoMAM (2026/3)? Your sequential training is outdated."

## 2. 技术摘要

### 2.1 形式化

整个 memory pipeline 形式化为 MDP：

```
State: (current_memory, current_input, conversation_history)
Action: 来自整个 multi-agent pipeline 的复合 action
   - construction_action: 写 / 更新 memory
   - retrieval_action: retrieve 哪些 fragment
   - answer_action: 生成 final answer
Reward: end-to-end task reward (e.g., LongMemEval QA F1)
```

整个流程像一个长 trajectory，但中间多个 agent 各负责一段。

### 2.2 算法：joint GRPO with adaptive credit

CoMAM 用 GRPO 训整条 trajectory，但加上了 **adaptive credit modeling**：

```python
def adaptive_credit_grpo(rollouts):
    """每个 agent action 拿到的 reward 不是平均分配，
       而是 learnable 的"""
    # 标准 GRPO 给的是 trajectory-level advantage
    A = group_relative(trajectory_rewards)
    
    # CoMAM 在 trajectory 内做"重新分配"
    # 每个 agent action 的 credit = A * learned_attribution_weight
    for traj in rollouts:
        for action in traj.actions:
            attribution = predict_attribution(action, traj)  # learnable
            action_credit = A * attribution
```

具体 attribution model 是什么 CoMAM 用了什么 architecture，需要看论文 detail（待精读）。但大致是 attention-style or learnable per-agent weight。

### 2.3 模型

单 backbone Qwen2.5-7B，用 role-specific prompts 区分 agents。这是和 Memory-R2 一致的设计——**parameter-efficient co-learning**。

### 2.4 评测

- PersonaMem (32K / 128K / 1M token context)
- LongMemEval (multi-session)

### 2.5 结果

CoMAM vs Memory-R1：**+8.5% to +16.7%** 在 PersonaMem 上。

---

## 3. CoMAM 自己承认的局限

来自论文 limitations section：

1. **简化 agent design**：只 3 个 core agents（construction / retrieval / answer），实际 production 可能更多
2. **cross-agent credit modeling 简化**：agents 间 local reward 的 relationship 没显式建模——只是 learnable weight
3. **隐私风险未处理**：存储用户敏感信息没有安全机制

## 4. 对 SpeakerMem-R1 v2 的具体借鉴

### 4.1 直接借鉴：joint training

我们 v2 必须明确：

```python
def speakermem_v2_training():
    # 不是先训 construction 冻结再训 retrieval，而是同时训
    
    one_backbone = Qwen2_5_7B_LoRA()
    
    role_prompts = {
        'construction': "You are a memory writer. Decide what to store...",
        'retrieval': "You are a memory retriever. Select relevant fragments...",
        'attribution': "You are a speaker attributor. Tag this fact with speaker...",
        'answer': "You are an assistant. Answer using retrieved memories...",
    }
    
    for batch in dataloader:
        # 一条 trajectory 内调用所有 4 个 roles
        traj = rollout(one_backbone, role_prompts, batch)
        
        # 算 end-to-end reward (DeltaMem-style + outcome)
        reward = combined_reward(traj)
        
        # joint update
        loss = LoGo_GRPO_loss(traj, reward)
        loss.backward()
```

### 4.2 拓展：speaker-aware adaptive credit

CoMAM 的 adaptive credit 是 global learnable weight。多方场景下 credit 应该 **per-speaker**：

```python
def speaker_aware_adaptive_credit(traj, speakers):
    """每个 agent action 的 credit，按它影响的 speaker 加权"""
    A_global = group_relative(traj.rewards)
    
    for action in traj.actions:
        # action 影响的 speaker 集合
        affected_speakers = action.speaker_id_list
        
        # 每 speaker 的"重要性权重"是 learnable
        per_speaker_weight = sum(
            speaker_importance[s] for s in affected_speakers
        )
        
        action_credit = A_global * per_speaker_weight
```

speaker_importance 可以是 learnable embedding（per-speaker scalar），也可以是 context-dependent（attention from conversation context to speaker）。

### 4.3 与 DeltaMem 的 reward 相容

CoMAM 用 outcome reward，没用 dense state reward。我们 v2 既用 joint training（CoMAM）又用 dense reward（DeltaMem）+ LoGo（Memory-R2）。

这三者**完全相容**：
- CoMAM 决定**谁训什么**（all jointly）
- DeltaMem 决定**reward 是什么形状**（dense state diff）
- Memory-R2 决定**advantage 怎么算**（local + global）

这就是 SpeakerMem-R1 v2 的**三大算法支柱**。

### 4.4 隐私层（CoMAM 没做的）

CoMAM 承认没处理隐私。我们 v2 加 **R_leak** 项（cross-speaker leakage penalty）正好补这个空白。如果走 idea 2 (GroupPolicy) 路线还可以更深入做 access control。

## 5. CoMAM vs Memory-R2 对比

| 维度 | CoMAM | Memory-R2 |
|------|-------|-----------|
| 核心创新 | joint multi-agent training | LoGo local rerollout |
| Reward 类型 | trajectory outcome | trajectory outcome |
| Credit 机制 | adaptive per-agent weight | local rerollout group-relative |
| Setting | dyadic | dyadic |
| 时间 | 2026/3 | 2026/5 |

**两者正交可结合**：joint training（CoMAM 框架）+ LoGo 算法（Memory-R2 替换标准 GRPO）+ DeltaMem dense reward。

## 6. Related Work 写法

> **Joint Training of Memory Agents.** CoMAM [cite] is the first to formalize the multi-agent memory pipeline as a joint MDP and train construction, retrieval, and answer agents end-to-end via GRPO with adaptive credit assignment, achieving 8.5-16.7% gains over sequentially-trained Memory-R1. However, CoMAM operates exclusively in dyadic settings, treats agents as task-defined rather than speaker-aware, and does not address privacy. Our SpeakerMem-R1 inherits CoMAM's joint training paradigm but generalizes it to **speaker-grounded multi-agent training**, where each agent action is conditioned on (speaker_id, audience_set) and credit is assigned per-speaker via a learned speaker-importance weighting.

## 7. 整体设计哲学

CoMAM 的洞察：**"Memory operations are coupled. Train them together."**

我们 v2 的扩展：**"In multi-party, memory operations are coupled across both agent roles AND speaker identities. Train them all together, with per-speaker credit."**

## 9. V3 技术细节补充（2026-06-02 调研更新）

### 9.1 Adaptive Credit Modeling 实现方式（已解决）

CoMAM 的 adaptive credit 是**基于 ranking consistency 的 per-agent scalar weight**：

```python
# 核心机制：local reward 与 global reward 的 Spearman/Kendall 排名相关性

def adaptive_credit_weight(local_rewards, global_rewards) -> float:
    """
    rank_consistency = spearman(rank(local_rewards), rank(global_rewards))
    alpha_i = normalize(rank_consistency) ∈ [0,1]
    """
    rank_consistency = spearman_correlation(local_rewards, global_rewards)
    return normalize_to_01(rank_consistency)

# agent 最终优化目标
agent_loss = local_reward + alpha_i * global_reward
```

**重要修订**：之前推测可能是 attention-style，实际是基于统计排名相关性的自适应系数，**不需要梯度传播额外 overhead**，实现相对简单。

### 9.2 三类 Agent 架构明确

| Agent | 核心职责 | Local Reward |
|-------|---------|-------------|
| Extraction Agent | 细粒度 memory 条目构建 | information coverage |
| Profile Agent | 粗粒度 user profile 摘要 | profile accuracy |
| Retrieval Agent | 记忆检索 + 问答 | QA accuracy（=global reward）|

**顺序式 MDP**：上游输出是下游状态输入；joint RL 联合优化。

### 9.3 消融结论

自适应信用 (CoMAM) > 均匀全局信用 (AT-GRPO) > 独立优化

→ **最大贡献是 adaptive credit，而非 joint MDP 结构本身**（这是很重要的 insight，意味着 credit 设计比架构更关键）

### 9.4 对 SpeakerMem 的扩展方案

基于 rank consistency 的 speaker-level credit：

```python
def speaker_credit_weight(speaker_id, per_speaker_local_rewards, global_reward):
    """
    把 CoMAM 的 per-agent credit 扩展到 per-speaker：
    "哪个说话人的 memory operation 对全局 QA 贡献更大？"
    """
    speaker_locals = list(per_speaker_local_rewards.values())
    global_list = [global_reward] * len(speaker_locals)
    return spearman_correlation(speaker_locals, global_list)
```
