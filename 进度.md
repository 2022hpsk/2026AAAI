### 2026-06-02 （第 3 轮 V3 迭代 - 第 2 步）

**V3 调研结果**（两个并行 Agent 返回后整合）：

**发现1：三大 benchmark 均无训练集（关键障碍已确认）**
- GroupMemBench：纯评测集，无训练 split
- EverMemBench：纯评测集，无训练 split（GitHub 开放评测数据）
- SocialMemBench：纯评测集，无训练 split
- **影响**：SpeakerMem-R1 训练数据需完全合成（已设计 200 条合成方案）

**发现2：三篇竞品均无公开代码（截至 2026/6/2）**
- CoMAM (2603.12631)：无 GitHub
- DeltaMem (2604.01560)："code upon acceptance"
- Memory-R2 (2605.21768)：无 GitHub
- **影响**：需从零实现，但也意味着 reviewer 更难 reproduce 竞品 baseline

**发现3：CoMAM adaptive credit = rank consistency scalar（不是 attention）**
- 实现比想象简单：`alpha_i = spearman_rank_correlation(local_rewards, global_rewards)`
- 直接可扩展为 per-speaker credit

**发现4：Memory-R2 消融揭示 Fact Extraction 是最关键组件**
- 仅训练 Fact Extractor → F1=28.30（崩塌）
- 去掉 co-learning → F1=45.34（-4.33）
- 去掉 LoGo → F1=46.62（-3.05）
- **对 SpeakerMem 含义**：Speaker-Aware Fact Extraction 应成为最大卖点

**发现5：DeltaMem 使用 Qwen3-8B（建议我们对齐基座）**
- τ 敏感性：过小时 LoCoMo 性能下滑，推荐范围 0.5~0.7
- Local lexical fidelity 对 PersonaMem 贡献 +3 F1（必须保留）

**发现6：BM25 在 GroupMemBench 强的原因**
- 合成数据词汇一致性高（BM25 IDF 加权占优）
- Agent memory 系统在多方场景中引入额外噪音（entity merging 等）
- 绝对最高才 46%，"BM25 强" = 大家都弱
- **含义**：真正瓶颈是 memory writing quality，不是 retrieval
