# 30+篇LLM记忆/多轮对话/RL训练论文精细深度精读报告（含2026补充）

## TL;DR
- 将30+篇论文归为四大族群：Agentic记忆系统（A-MEM/Mem0/LD-Agent/MemoryBank/Generative Agents/新增MemGPT）、RL记忆训练（Memory-R1/Mem-α/MEM1/MemAgent/MemSearcher/AgeMem/动态压缩记忆/新增ExpGraph）、反思自演化（Reflexion/SCoRe/ReasoningBank）、Benchmark/数据集（GroupMemBench/LongMemEval/LOCOMO/新增SocialMemBench/EverMemBench/MemGym等）。核心结论：记忆管理正从"启发式静态pipeline"转向"用RL端到端学习ADD/UPDATE/DELETE/检索/压缩策略"，而评测端则从单轮exact-match转向多会话、LLM-as-judge、记忆变更（knowledge update）压力测试。
- 技术上有两条主线：一是"外部记忆库+检索"（schema=结构化note+embedding，检索打分普遍为recency×importance×relevance或带时间衰减e^(−t/τ)），二是"上下文内压缩记忆"（固定大小internal state/memory token，靠RL保持近似常量上下文）。RL类几乎全部采用GRPO/PPO/DAPO变体、以最终答案正确性为outcome reward。
- 多角色/多方对话记忆是明确的前沿空白：GroupMemBench最强系统平均准确率仅46.0%（knowledge update仅27.1%、term ambiguity仅37.7%），SocialMemBench显示多种记忆框架仅0.12–0.18、低于未压缩检索基线0.345，EverMemBench即便oracle证据多跳归因也仅26%；说明现有记忆摄取会抹除群体记忆所依赖的结构与词汇特征，角色归因与版本更新仍是核心瓶颈。

## Key Findings
1. **记忆操作的"可学习化"是2025年最大趋势**：Memory-R1、Mem-α、AgeMem均把"存什么/何时更新/如何结构化"作为RL策略输出，而非硬编码规则。Memory-R1仅用152个QA对训练，用LLaMA-3.1-8B-Instruct即在LOCOMO上相对最强基线Mem0实现"overall F1 +48%、BLEU-1 +69%、LLM-as-a-Judge +37%"。
2. **上下文常量化压缩**（MEM1、MemAgent、MemSearcher、动态压缩记忆）通过迭代覆写internal state，使峰值token近似常量。MemAgent官方称可"从32K文本训练的8K上下文外推到3.5M QA任务、性能损失<5%、512K RULER达95%+"（RL-MemAgent-14B在3.5M token任务降幅<5.5%，7B约11%）。
3. **Benchmark的难度来自"非平稳性"**：LongMemEval的knowledge-update、GroupMemBench的knowledge update(27.1%)与term ambiguity(37.7%)、MultiChallenge的versioned editing都暴露出模型难以处理"信息被更新/撤销"；EverMemBench进一步显示缺少显式版本语义时，时间推理会系统性失效。
4. **LLM-as-judge已成多轮评测标配**：MT-Bench确立GPT-4判官与人类>80%一致，MT-Bench-101、MultiChallenge、PERSONAMEM、LOCOMO、LongMemEval均采用；但exact-match/ROUGE仍主导长上下文benchmark（LongBench、InfiniteBench）。
5. **多角色记忆是研究洼地**：GroupMemBench、SocialMemBench、EverMemBench、MuPaS、SHARE、MOOM分别从评测、生成、共享记忆、角色扮演长对话切入，但缺乏统一的per-speaker信念追踪与受众自适应建模；SocialMemBench中现有记忆框架0.12–0.18、低于未压缩检索0.345，EverMemBench即便oracle证据多跳归因也仅26%，显示角色归因与群体规范/例外分离仍是硬瓶颈。
6. **评测正在从聊天走向agentic执行**：MemGym将工具对话/深度检索/编码/网页操作统一到memory-isolated评分，并给出轻量MemRM奖励模型，适合作为RL记忆训练的低成本评价闭环。

### 第一部分：Agentic 记忆系统

#### 1. A-MEM: Agentic Memory for LLM Agents (arXiv:2502.12110, Xu et al., 2025)
**架构与数据结构**：A-MEM受Zettelkasten卡片盒笔记法启发，构建可动态互联的记忆网络。每条新交互被构造为一个"原子笔记"(atomic note)，其schema包含多个结构化属性：原始交互内容、时间戳(timestamp)、LLM生成的上下文描述(contextual description)、关键词(keywords)、标签(tags)，以及用于相似度匹配的嵌入向量(embedding)。生产实现使用ChromaDB作为向量存储。
**核心算法（三阶段）**：(1) **Note Construction（笔记构造）**：新记忆到达时，LLM自动生成summary/keywords/tags等属性。(2) **Link Generation（链接生成）**：基于嵌入相似度从历史记忆库检索候选近邻，再由LLM判断是否建立显式链接（meaningful connection），形成图状结构。(3) **Memory Evolution（记忆演化）**：新记忆并入后，可触发对相关历史笔记的上下文表示与属性的更新——即历史笔记的描述/标签会随新知识被改写。
**检索**：查询被分解为关键词，用嵌入相似度做top-k检索；相比full-context可大幅降低token使用。
**实验**：在LOCOMO与多方长对话QA数据集上，在六个基础模型上均优于SOTA基线。
**局限**：记忆演化会"静默改写"高影响力历史记忆，带来版本/审计与安全隐患；链接生成依赖LLM判断，成本与一致性受backbone影响。

#### 2. Mem0: Production-Ready AI Agents with Scalable Long-Term Memory (arXiv:2504.19413, Chhikara et al., 2025, ECAI 2025)
**架构（两阶段pipeline）**：(1) **Extraction Phase**：从持续对话中动态抽取salient信息（episodic/semantic/procedural三类记忆）。(2) **Update Phase**：将新事实与已有记忆做整合(consolidate)，决定ADD/UPDATE/DELETE。**Mem0g变体**用图记忆表示：以Neo4j作为底层图数据库，GPT-4o-mini负责实体/关系抽取与图更新，节点为实体、边为关系，以建模跨会话关系结构。
**关键数字与推理流程**：检索时只取与问题相关的记忆而非全历史。论文原文称Mem0在LLM-as-a-Judge指标上相对OpenAI Memory有"26% relative improvements"，Mem0g图记忆再高约2%整体分；且"Mem0 operates with 91% lower response times than full-context approaches"，p95延迟约1.44s，每用户可处理约一万条记忆、更新延迟亚100ms。
**实验**：在LOCOMO上对比六类基线（memory-augmented如MemGPT/MemoryBank/A-Mem、不同chunk与k值的RAG、full-context、开源LangMem、专有OpenAI Memory、Zep）。Mem0在Single-Hop/Multi-Hop/Temporal上达SOTA。LOCOMO评测含10段长对话、每段约600轮、约26K token、平均约200问/段。
**局限**：LOCOMO评测设置受社区质疑（有报告称改动设置后Zep可反超Mem0约24%）；Mem0g复杂图结构相比Mem0增益有限。

#### 3. LD-Agent: Hello Again! LLM-powered Personalized Agent for Long-term Dialogue (arXiv:2406.05925, Hao Li et al., NAACL 2025)
**三模块架构**：(1) **事件记忆感知**（长期+短期记忆库）；(2) **动态人格抽取**（双向建模user与agent人格）；(3) **响应生成**。
**记忆数据结构（精确）**：长期记忆 M_L = {φ(t_j, o_j)}，存储发生时间t与事件摘要o，由文本编码器φ（MiniLM）编码；短期记忆 M_S = {(t_i, u_i)}，为当前会话带时间戳的话语；人格库 P_u / P_a，由LoRA微调的抽取器更新，无特征时输出"No Trait"。
**检索打分公式（核心）**：主题重叠 s_top = ½(|V_q∩V_k|/|V_q| + |V_q∩V_k|/|V_k|)；**时间衰减系数 λ_t = e^(−t/τ)**；总分 **s_overall = λ_t·(s_sem + s_top)**（语义相关+主题重叠，按时间衰减重加权）；阈值γ过滤，无命中返回"No relevant memory"。当会话时间间隔超过阈值β时，短期对话被摘要写入长期记忆并清空缓存。
**训练**：事件摘要器在DialogSum微调，人格抽取器在MSC衍生数据微调；ChatGLM3-6B，lr 5e-5，batch 4，3 epoch，单张A100，全模块LoRA。
**实验**：数据集MSC、Conversation Chronicles(CC)（均5会话、约50轮/样本、4000/500/501划分），多方场景用Ubuntu IRC。指标BLEU-N/ROUGE-L/METEOR + 人格分类ACC + 人工评测。相对前SOTA HAHT在BlenderBot+LDA约提升3.3–3.7 BLEU-2。

#### 4. MemoryBank: Enhancing LLMs with Long-Term Memory (AAAI 2024, Zhong et al., arXiv:2305.10250)
**存储结构**：按时间顺序的多轮对话记录（带时间戳）+ 分层摘要的事件summary + 用户画像(personality/mood)。每条记忆m预先用编码器E(·)编码为表示h_m。
**三组件**：记忆存储、记忆检索（基于稠密嵌入余弦相似度的FAISS式检索）、记忆更新。
**艾宾浩斯遗忘曲线公式（核心）**：保持率 **R = exp(−t/S)**，t为距上次回忆的天数，S为记忆强度；每次被回忆则强化S（增大、延长衰减），模拟间隔重复；未回忆的记忆衰减并可被遗忘。
**SiliconFriend**：基于MemoryBank的AI伴侣聊天机器人，在约38k心理咨询对话上微调以增强共情；支持ChatGPT、ChatGLM-6B、BELLE，中英双语。
**评测**：定性（真实用户对话）+ 定量（ChatGPT模拟多样用户生成约10天长期对话），评估记忆召回、共情、人格理解。

#### 5. Generative Agents: Interactive Simulacra of Human Behavior (UIST 2023, Park et al., arXiv:2304.03442)
**三组件**：记忆流(memory stream)、反思(reflection)、规划(planning)。
**记忆流结构**：记忆对象列表，每条为自然语言观察，带创建时间戳与最近访问时间戳。
**检索打分（核心公式）**：score = α_recency·recency + α_importance·importance + α_relevance·relevance，实现中三权重均为1（min-max归一到[0,1]）。recency = 按游戏内小时数的指数衰减，**衰减因子0.995**；importance = LLM生成的1–10整数（平凡=1，深刻=10）；relevance = 查询嵌入与记忆嵌入的余弦相似度。
**反思**：当近期事件importance之和超过阈值（约150）时周期性触发（约2–3次/天），合成更高层推断，形成反思树（观察为叶、反思为高层节点，可递归反思）。
**规划**：递归分解 日→小时→5–15分钟动作计划，遇新观察则react/replan。
**环境**：Smallville沙盒，25个agent；消融显示完整架构在可信度上优于去反思/去规划/去记忆变体。

### 第二部分：RL 记忆训练

#### 6. Memory-R1 (arXiv:2508.19828, Yan et al., 2025)
**双agent架构**：(1) **Memory Manager**：每轮对话后决定结构化操作{ADD, UPDATE, DELETE, NOOP}，动态更新外部记忆库；(2) **Answer Agent**：对每个问题先检索最多60条候选记忆，蒸馏(distill)为最相关子集，再推理生成答案。
**RL设计**：两agent均用PPO或GRPO微调，奖励信号仅为QA正确性（outcome-driven，无需人工标注记忆操作标签）。GRPO通过组内相对优势估计避免训练value网络。
**训练数据效率**：仅152个QA对+对应temporal memory bank即可训练。
**实验结果（abstract/Table 1原文）**：LLaMA-3.1-8B-Instruct上，Memory-R1-GRPO相对最强基线Mem0："improves overall F1 by 48%, BLEU-1 by 69%, and LLM-as-a-Judge by 37%"；在Qwen-2.5-7B上有类似增益。泛化到三个benchmark（LOCOMO、MSC、LongMemEval）与3B–14B多尺度。
**关键洞见**：vanilla系统会把"先后领养两只狗"误判为矛盾而DELETE+ADD碎片化记忆；RL训练的Manager学会用单个UPDATE合并事实（abstract配图示例：Answer Agent把60条检索记忆蒸馏到相关的"Andrew adopted 2 dogs named Buddy and Scout"并正确答"2 dogs"）。

#### 7. Mem-α (arXiv:2509.25911, Wang et al., 2025)
**问题**：现有记忆agent依赖预定义指令与工具，LLM难以判断存什么/如何结构化/何时更新，尤其记忆系统复杂时。
**方法**：RL框架训练agent通过交互与反馈管理复杂记忆系统（含多种记忆类型与工具）。因无法从任何现有模型获得可靠监督信号（连GPT-4o都难正确选择记忆更新工具，小模型则被复杂工具集压垮），直接对下游任务表现做端到端优化（trial-and-error发现最优记忆策略，无需ground-truth构造trace）。
**训练数据**：专门构造的训练集，覆盖多样多轮交互模式并配综合评测问题；复用MemoryAgentBench处理后的数据。
**实验**：在多尺度模型上证明RL习得的记忆策略优于prompt-based基线；代码开源(wangyu-ustc/Mem-alpha, Apache-2.0)。

#### 8. MEM1 (arXiv:2506.15841, Zhou et al., 2025, ICLR 2026)
**核心思想**：在共享表示空间内统一reasoning与memory consolidation，无需额外模块。每轮维护紧凑的共享internal state (IS)。
**算法流程**：每步agent基于IS_t决定下一动作（向环境发query_t或直接answer）；收到observation info_t后，将(IS_t, query_t, info_t)整合(consolidate)为IS_{t+1}，并**剪枝旧状态只保留最新IS**，从而上下文近似常量。
**RL训练**：端到端RL（verifiable reward，最终答案正确性）；专门的attention masking方案适配动态记忆；自定义multi-turn rollout做迭代压缩；rollout trajectory truncation；在policy update阶段用information mask屏蔽非模型生成token。提出可扩展task augmentation把单目标QA合成多目标任务。
**结果**：16目标设置下，MEM1准确率超过Qwen2.5-14B-Instruct，而仅用其27.1%峰值token和29.3%推理时间。RL显著优于SFT（SFT在>6目标时崩溃）；加format reward加速收敛但损害最终性能。涌现行为：并发多问题管理、动态焦点转移、记忆整合、query精炼、自我验证、子目标分解。

#### 9. MemAgent (arXiv:2507.02259, Yu et al., 2025)
**工作流**：模仿人类读长文——将文档切分为多个chunk，LLM迭代处理每个chunk，用**覆写(overwrite)策略**更新固定大小的token级记忆，最后基于记忆生成答案，实现线性时间复杂度与有限上下文窗口处理任意长输入。
**训练算法**：扩展DAPO为**Multi-Conv DAPO**，支持独立上下文的多轮(多conversation)生成训练；基于verl实现，定义Multi-Conv工作流/数据集/配置统一接口。RLVR（可验证奖励）。
**结果（abstract原文）**："extrapolate from an 8K context trained on 32K text to a 3.5M QA task with performance loss < 5% and achieves 95%+ in 512K RULER test"；项目页补充RL-MemAgent-14B在3.5M token任务性能下降<5.5%、7B约11%。

#### 10. MemSearcher (arXiv:2511.02805, Yuan et al., 2025)
**工作流**：搜索agent的backbone LLM兼作记忆管理器。每轮融合用户问题与当前memory生成reasoning trace，执行搜索动作（输出\boxed{answer}或发起搜索调用），并**覆写更新memory**——读入observation o_i，纳入有助回答的新信息，同时保留前一memory m_{i-1}中的相关细节，memory受预定义最大长度约束，使上下文短而稳定。
**Multi-Context GRPO（核心）**：每条trajectory由不同上下文下的多个conversation组成；将trajectory级advantage传播到每个conversation，并将每个conversation视为独立优化目标。
**训练与结果**：用与Search-R1相同数据，从Qwen2.5-3B/7B-Instruct从零训练；7个公开知识获取benchmark上分别相对提升11%/12%；token数随轮数近似常量，优于ReAct式历史拼接。

#### 11. AgeMem / Agentic Memory: Unified Long-Term and Short-Term Memory Management (arXiv:2601.01885, Yu et al., 2026)
**架构**：把LTM+STM管理直接整合进agent策略（而非独立模块），将记忆操作暴露为基于工具的动作（六个核心工具：LTM的store/update/discard，STM的retrieve/summarize/remove），由LLM agent自主调用。基于AgentScope框架。
**RL建模**：状态s_t = 对话上下文+STM+LTM(M_t)，动作a_t来自混合空间（自然语言生成+记忆操作），策略π_θ最大化E_{τ∼π_θ}[R(τ)]。
**三阶段渐进RL**：阶段1 LTM构建（在闲聊中识别salient信息）；阶段2 干扰下的STM控制（保留LTM、重置短上下文、喂误导/无关信息）；阶段3 组合长程推理。用step-wise GRPO处理稀疏/不连续奖励。
**奖励函数**：R(τ)结合任务完成奖励R_task + 记忆质量项 + 上下文管理奖励R_context（效率）+ 惩罚项。
**结果**：五个长程benchmark（含HotpotQA）上优于MemAgent、Mem1式等强基线，任务表现更好、LTM质量更高、平均token/上下文使用更低。（注：各benchmark精确数字来自论文Table 2，公开摘要未完整给出。）

#### 12. 动态压缩记忆 Dynamic Long Context Reasoning over Compressed Memory via End-to-End Reinforcement Learning (arXiv:2602.08382, Chen et al., 2026, 框架名 LycheeMemory)
**方法**：认知启发的chunk-wise压缩+选择性记忆召回。共享base LLM Φ上三个LoRA模块：(1) **Compressor Φ_comp**：将文档D切为K个固定大小(如4096 token)chunk，每chunk C_i→紧凑latent θ_i，用z_i = w/α_i个可训练memory token（α_i为压缩比），产出压缩KV-cache表示；(2) **Gate Φ_gate**：动态相关性过滤，选择相关压缩记忆块（单独训练为分类器）；(3) **Reasoner Φ_reason**：迭代处理被选块，更新明文工作记忆求解任务。
**训练**：Compressor+Reasoner端到端RL联合优化；Gate作为分类器训练。
**结果**：多跳benchmark(RULER-HQA)上具竞争力；上下文7K→1.75M外推；相比MemAgent峰值GPU内存降至约1/2、推理加速约6×；零样本泛化到摘要。

### 第三部分：反思 / 自演化

#### 13. Reflexion (NeurIPS 2023, Shinn et al., arXiv:2303.11366)
**框架**：用语言反馈而非权重更新来强化语言agent。三组件：**Actor**（生成文本/动作）、**Evaluator**（对输出打分，可启发式或LLM）、**Self-Reflection模型**（生成口头强化线索）。agent对任务反馈做口头反思，将反思文本存入**episodic memory buffer**，在后续trial中改进决策。
**流程**：执行trajectory → Evaluator打分（标量或自然语言反馈，外部或内部模拟）→ Self-Reflection生成"为何失败/如何改进"的文本 → 写入memory → 下一trial把反思作为上下文。
**结果（abstract原文）**："Reflexion achieves a 91% pass@1 accuracy on the HumanEval coding benchmark, surpassing the previous state-of-the-art GPT-4 that achieves 80%"。代码开源(noahshinn/reflexion)，支持决策(ALFWorld)、推理(HotpotQA)、编程三类；可配置REFLEXION与LAST_ATTEMPT_AND_REFLEXION策略。
**局限**：依赖可靠的reward/evaluator信号；记忆为episodic buffer，跨任务迁移有限。

#### 14. SCoRe (arXiv:2409.12917, Kumar et al., DeepMind, ICLR 2025)
**目标**：用完全自生成数据的多轮在线RL提升LLM自我纠错能力，解决SFT的分布失配与"minimal edit"问题。
**两阶段训练（核心）**：**Stage I（初始化）**：训练模型在保持第一回答分布接近base model（KL约束）的同时优化第二回答的正确性，解耦两次尝试、避免行为崩溃。**Stage II（联合优化+reward shaping）**：联合优化两次尝试奖励，并加**reward shaping项 α·(r(y2)−r(y1))**奖励"自我纠错的进步"而非单纯第二次正确，防止坍缩为"直接给最优首答+微调"。
**结果（abstract原文）**：在Gemini 1.0 Pro/1.5 Flash上，SCoRe相对base model自我纠错分别在MATH与HumanEval上提升"15.6% and 9.1% respectively"。消融显示去Stage I或去reward shaping分别使Δ(t1,t2)降2.2%/1.8%。

#### 15. ReasoningBank (arXiv:2509.25140, Ouyang et al., Google, 2025)
**记忆条目schema**：每条memory item含title（标题）、one-line description（一句话描述）、content（可操作原则：启发式/检查项/约束）。存储策略级(strategy-level)经验而非原始日志或仅成功流程。
**闭环流程**：retrieve（嵌入检索top-k相关item注入为system guidance，默认k=1）→ inject → execute → judge → distill（从成功提取validated策略，从失败提取反事实信号与陷阱，每条trajectory抽多个item）→ consolidate（简单addition并入）。
**MaTTS（Memory-aware Test-Time Scaling）**：并行/串行扩展test-time计算并与记忆协同。
**结果（原文）**：在WebArena、Mind2Web、SWE-Bench-Verified等benchmark上，相对存原始轨迹/仅成功流程的记忆机制，"outperform baselines in both effectiveness (up to 34.2% relative improvement) and efficiency (16.0% less interaction steps)"；MaTTS进一步放大增益。确立"记忆驱动的经验scaling"为新scaling维度。

### 第四部分：Benchmark / 数据集（数据格式重点）

#### 16. GroupMemBench (arXiv:2605.14498, Jingbo Yang et al., 2026)
**定位**：首个面向多方(multi-party)对话的LLM记忆benchmark，暴露三个被现有dyadic设置忽略的属性：(i)群体动态（超越拼接的一对一聊天）、(ii)speaker-grounded信念追踪（需per-user记忆建模）、(iii)受众自适应语言（Theory-of-Mind导致角色专属词汇）。
**数据合成pipeline**：graph-grounded合成产生可控reply结构的多方对话，每条消息以per-user persona与目标受众为条件；adversarial query pipeline把每个问题绑定到特定提问者，迭代搜索有挑战性的真实查询。
**任务类型（六类）**：multi-hop reasoning、knowledge update、term ambiguity、user-implicit reasoning、temporal reasoning、abstention。
**评测协议与关键结果（abstract原文）**："the strongest one reaches only 46.0% average accuracy, with knowledge update at 27.1% and term ambiguity at 37.7%, while a simple BM25 baseline matches or exceeds most agent memory systems"。
**数据格式（推断schema）**：样本含多方对话消息序列（每条消息带speaker_id、target_audience、persona条件）+ 绑定asker的query + category标签 + gold answer。
**局限**：合成数据，真实多用户部署与会话噪声未覆盖。

#### 17. MemoryAgentBench (arXiv:2507.05257, Hu/Wang/McAuley, UCSD, ICLR 2026)
**四核心能力（基于记忆科学/认知科学）**：accurate retrieval（准确检索）、test-time learning（测试时学习）、long-range understanding（长程理解）、conflict resolution（冲突解决，v1称selective forgetting/遗忘）。
**数据构造**：将现有长上下文数据集切成多个chunk增量喂给agent，模拟多轮信息累积；额外新建两个数据集——**EventQA**（评测accurate retrieval）与**FactConsolidation**（评测conflict resolution）。LRU用172K+ token长叙事的摘要查询实例化；TTL通过拼接大量带标注样本、读完后分类新item实现；CR用拼接的edit pairs、要求只按最新事实回答。
**评测对象**：商用记忆agent(Mem0/MemGPT)、把全输入当记忆的长上下文agent、RAG agent。统一框架（含memory-construction阶段与query阶段），消融chunk大小/retriever TopK/session长度。
**数据格式**：标准化构造prompt，分构造阶段与查询阶段的统一task模板。GitHub: HUST-AI-HYZ/MemoryAgentBench。
**结论**：现有方法无一掌握全部四能力。

#### 18. SHARE (ACL 2025 Long, Kim/Park/Chang, aclanthology 2025.acl-long.704, arXiv:2410.20682)
**定位**：从电影剧本构建的共享记忆感知开放域长期对话数据集，捕捉persona、个人事件、隐式可抽取的共享记忆(shared memory)。
**数据格式（字段结构）**：每段对话含两人的persona信息摘要、各自的个人事件(events)、以及对话中显式揭示与隐式可抽取的共享记忆；按五准则分类对话（1.Persona信息：性格/职业/年龄/教育/喜好食物/音乐/爱好/家庭等；2.个人事件；3.共享记忆等）。含memory update动作及标签表（见论文Table 8）。
**数据来源**：电影剧本（角色共指消解处理），LLM辅助抽取persona/事件，比众包或纯LLM摘要更经济完整。
**EPISODE框架**：基于SHARE，摘要persona与个人事件、从对话抽取共享记忆，用于响应生成。
**评测**：Amazon Mechanical Turk人工评测，从96个test集随机选25段对话（random seed 2025）、20标注者按closeness/engagingness/coherence评分（第3会话进行）；Share模型优于Noshare。
**规模**：96个test对话；用GPT-4o与GPT-3.5-turbo降低成本。

#### 19. MOOM (arXiv:2509.11860, Weishu Chen et al., 2025)
**定位**：首个面向超长角色扮演对话、基于文学理论的双分支记忆插件（plot发展 + character刻画为核心叙事元素）。
**双分支架构**：分支一在多时间尺度上总结plot conflicts（情节冲突）；分支二用预定义关键词构建并更新user的character profile（人物画像）。
**遗忘算法（核心）**：基于"竞争-抑制"(competition-inhibition)理论(Anderson 2003)，通过时间衰减与检索强化对相关信息优先、抑制低关联，模拟人类记忆，约束记忆容量、防止无限增长，保持灵活记忆容量与多样信息。
**数据集 ZH-4O（中文角色扮演长对话记忆数据集）**：收集标注者与LLM的角色扮演对话，由独立标注者标注关键信息及其在对话中的位置；共28段带记忆标注对话，平均每段约600轮。设计多组探测问题覆盖各类角色扮演信息。
**贡献**：首个围绕"故事元素"设计的长记忆抽取方法 + 遗忘算法 + ZH-4O基准。

#### 20. MuPaS (arXiv:2412.05342, Xiaoyu Wang et al., 2024)
**定位**：多方对话(MPD)监督微调框架，让单个LLM直接从多方对话学习，统一response生成与speaker，并刻画不同persona风格。
**训练方法（核心）**：与常规LLM训练类似，对多方对话文本前向传播得logits；**对每个角色计算微调loss时，把非活跃角色部分mask掉**（inactive role masking）。
**两训练策略（转为MPD模拟器）**：(1) **Speaker Predictor**（中心化架构，预测下一说话者）；(2) **Silence Switcher**（去中心化架构）。
**推理**：给定system prompt与历史话语生成下一话语，新话语可追加到对话末尾继续推理。
**实验**：在Friends数据集（按Yang et al. 2019划分）等上，达SOTA多方response、更高next-speaker预测准确率、更高人工与自动话语质量，对OOD场景也能合理生成。

#### 21. LongMemEval (arXiv:2410.10813, Wu et al., ICLR 2025)
**定位**：评测聊天助手长期交互记忆的五核心能力：信息抽取、多会话推理、时间推理、知识更新(knowledge update)、abstention（拒答）。500道人工精校问题（七种类型）嵌入可自由扩展的用户-助手聊天历史。
**数据格式（JSON字段，官方）**：每实例含 `question_id`（以`_abs`结尾为abstention题）、`question_type`（single-session-user / single-session-assistant / single-session-preference / temporal-reasoning / knowledge-update / multi-session）、`question`、`answer`、`question_date`、`haystack_session_ids`（历史session id列表，按时间戳排序）、`haystack_dates`（时间戳列表）、`haystack_sessions`（实际聊天历史内容）。
**两规模变体**：LongMemEval_S约115k token/问；LongMemEval_M约500 session、约1.5M token。另有oracle版（仅含evidence session）。
**评测协议**：LLM-as-judge（gpt-4o-2024-08-06），与人类专家一致率>97%。
**关键发现**：商用助手与长上下文LLM在持续交互记忆上准确率掉约30%（长上下文任务最高掉60%）。提出session decomposition、fact-augmented key expansion、time-aware query expansion等优化。

#### 22. LOCOMO (ACL 2024, Maharana et al., arXiv:2402.17753)
**定位**：评测very long-term对话记忆。机器-人pipeline生成：每个LLM agent被赋予独特persona与因果相连的事件时间线(temporal event graph)，配记忆与反思模块检索历史生成对话，支持图片分享/反应（多模态），人工标注者编辑保证长程一致性。
**数据规模**：50段高质量超长对话，平均约300轮、9K token、最多35会话（部分文献版本为600轮/16K token/32会话）；约9倍于MSC长度。7,500+人工QA对。
**任务与评测**：(1)**问答**（5类：single-hop、multi-hop、temporal、open-domain、adversarial），用DRAGON检索；(2)**事件摘要**（事件图重构）；(3)**多模态对话生成**（下一轮生成）。
**数据格式（推断）**：每段含speaker personas、event graph、多session对话（带时间戳、可含图片caption）、QA对（含category与evidence标注）。
**结论**：LLM难以理解长程叙事与时间/因果动态；长上下文LLM与RAG有改善但远逊人类。

#### 23. PERSONAMEM (COLM 2025, Jiang et al., arXiv:2504.14225, "Know Me, Respond to Me")
**定位**：评测LLM对动态用户画像的追踪与个性化响应。用户在多session中跨多主题（食物推荐、旅行规划、心理咨询等）与chatbot交互，偏好随时间演化。
**数据格式**：围绕连贯的user persona构建（同一persona_id下不同topic共享persona），含演化的偏好；提供QA对、对应context、meta data；问题为in-situ用户query类型（细粒度）。HuggingFace发布QA对+context+metadata。
**对比基准**：相比LOCOMO/LongMemEval（聚焦通用记忆检索）与PrefEval（先生成偏好再插入随机上下文），PERSONAMEM所有对话围绕连贯persona，更贴近真实user-chatbot对话。
**评测**：多选题(MCQ)；后续v2(2512.06688)加开放式+LLM-as-judge（三个GPT-5-Chat打分聚合），并用RFT训练Qwen3-4B超过GPT-5、agentic memory仅用2k-token记忆达55%准确率（16×更少token）。
**结论**：前沿模型仍难处理隐式个性化。

#### 24. MultiChallenge (ACL 2025, Sirdeshmukh et al., Scale AI, arXiv:2501.17399)
**四挑战类别**：(1)Instruction Retention（跨轮维持指令）；(2)Inference Memory of User Information（回忆/连接先前轮用户信息）；(3)Reliable Versioned Editing（协作编辑中适应演化指令）；(4)Self-Coherence（避免矛盾/谄媚）。四类均需同时做指令遵循+上下文分配+in-context推理。
**评测协议**：**LLM-as-judge配实例级rubric**（每段对话有自己的pass/fail评分标准），与资深人类评审取得fair一致。话题涵盖旅行规划、技术文档、专业沟通等。约273段人工精心构造对话。
**关键结果**：所有前沿模型<50%（尽管在旧多轮benchmark近满分）。论文v1：Claude 3.5 Sonnet(2024-06) 41.4%最高；Scale榜单：o1 44.93%（后续榜单GPT-5约69.6%、平均约47.8%）。

#### 25. MT-Bench-101 (ACL 2024, Bai et al., arXiv:2402.14762)
**三层层级分类法**：第一层3能力（Perceptivity感知、Adaptability适应、Interactivity交互）；第二层7细分能力；第三层**13个细粒度任务**（如Context Memory、Anaphora Resolution、Separate Input、Topic Shift、Content Confusion、Format Rephrasing、Proactive Interaction等）。
**规模**：**1,388段多轮对话、4,208轮**，覆盖13任务，基于真实多轮对话数据分析构建。
**评测**：GPT-4判官，每轮按1–10打分配任务专属评分指南；用**取最低分**metric（对话得分=最低轮分，惩罚任何失败点）。评测21个流行LLM。
**关键发现**：模型在适应性与交互性上有缺陷；RLHF等对齐技术与chat专用设计未明显改善多轮能力；不同任务跨轮趋势各异。

#### 26. MINT (ICLR 2024, Wang et al., arXiv:2309.10691)
**定位**：评测LLM在多轮交互中使用工具与语言反馈的能力。
**数据构造**：复用8个既有数据集（推理、代码生成、决策），先收集29,307实例，再分层子采样为**586个挑战性实例**（保持correct:incorrect比例）。
**工具使用**：LLM通过**生成并执行Python代码**调用工具。
**反馈**：自然语言用户反馈由**GPT-4(gpt-4-0613)模拟**以保证可复现/可扩展。
**评测指标**：Success Rate（任务专属正确性），作为交互轮数的函数；两轴——(a)工具增强求解（成功率vs工具轮数，k-turn预算=5）、(b)反馈增益（加GPT-4 NL反馈的成功率提升）。
**关键结果**：评测20个LLM（4闭源+16开源）；每多一轮工具使用绝对+1–8%；加NL反馈绝对+2–17%；单轮强≠多轮强。成本约$100/模型。

#### 27. MT-Bench & Chatbot Arena (NeurIPS 2023, Zheng et al., arXiv:2306.05685)
**MT-Bench**：80道问题、8类别（写作、角色扮演、信息抽取、推理、数学、编程、STEM、人文社科），每题2轮。
**Chatbot Arena**：众包匿名两两对战平台，用**Elo评分**排名。
**LLM-as-judge**：GPT-4判官与人类偏好**>80%一致**，与人-人一致水平相当。三种判分模式：pairwise比较、single-answer打分、reference-guided。
**偏差**：position bias（偏好第一个答案，靠交换位置缓解）、verbosity bias（偏好更长答案）、self-enhancement bias（偏好自身输出）、数学/推理判分能力有限。
**数据**：MT-bench含6模型答案、58专家标注者约3K票；Arena 3K单轮票（采样自30K）、2114众包评审。开源80问+3K专家票+30K对话。

#### 28. MSC / Beyond Goldfish Memory (ACL 2022, Xu/Szlam/Weston, arXiv:2107.07567)
**数据集**：Multi-Session Chat(MSC)，首个带session间时间间隔（小时-天）的多会话开放域数据集，人-人众包采集，建于PersonaChat（session 1=PersonaChat），每对话最多5会话，含双方persona记录与时间间隔元数据。（LD-Agent用：4000/500/501划分、约50轮/对话。）
**模型（核心）**：最佳方法=**summarization-based memory模型**——encoder-decoder把先前session摘要为记忆并读取；也含检索增强方法。两者在自动指标+人工评测上均优于标准长上下文encoder-decoder Transformer。
**结论**：短对话模型在长期设置下失效。

#### 29. LongBench (ACL 2024, Bai et al., arXiv:2308.14508)
**组成**：21数据集/6任务类别，中英双语。类别：单文档QA、多文档QA、摘要、few-shot学习、合成任务、代码补全。任务分布：14英文+5中文+2代码=21。
**平均长度**：英文6,711词，中文13,386字。
**数据格式（JSON字段）**：统一格式含 `input`（查询/问题）、`context`（长文档）、`answers`（列表）、`length`、`dataset`、`language`、`all_classes`（分类任务用）、`_id`。
**评测指标（per-task自动）**：QA用F1；摘要/中文用ROUGE-L；代码补全(LCC、RepoBench-P)用Edit Sim；few-shot分类(TREC、LSHT)用Accuracy；合成任务(passage retrieval/count)用Exact Match与检索准确率。**用exact-match/规则指标，非LLM-judge**。
**结果**：GPT-3.5-Turbo-16k在受测中最佳但长上下文退化；scaled位置编码+长序列微调有帮助；检索压缩对弱模型有帮助但逊于真长上下文模型。

#### 30. InfiniteBench / ∞Bench (ACL 2024, Zhang et al., arXiv:2402.13718)
**规模**：首个平均长度>100K token的benchmark；12任务；共3,946样本；中英双语；合成+真实；5领域（检索、代码、数学、小说、对话）。
**JSON schema（精确，HF）**：`id`:int64、`context`:string、`input`:string、`answer`:Sequence(string)、`options`:Sequence(string)；每任务存为.jsonl。
**12任务（名|上下文|样本数|平均输入token）**：En.Sum|fake book|103|171.5k；En.QA|fake book|351|192.6k；En.MC|fake book|229|184.4k；En.Dia|script|200|103.6k；Zh.QA|new book|175|2068.6k；Code.Debug|394|114.7k；Code.Run|400|75.2k；Math.Calc|50|43.9k；Math.Find|350|87.9k；Retrieve.PassKey|590|122.4k；Retrieve.Number|590|122.4k；Retrieve.KV|500|89.9k。
**指标**：多数任务用Accuracy；Zh.QA与En.QA用ROUGE F1；En.Sum用rougeLsum。规则/exact-match，无LLM-judge。
**结果**：GPT-4领先（PassKey 100%、Retrieve.KV 89%、En.MC 67%、Math.Find 60%）；Math.Calc所有模型<5%。

### 第五部分：新增文献补充（2026与关键缺口）

#### 31. MemGPT: Towards LLMs as Operating Systems (arXiv:2310.08560, Packer et al., 2023/2024)
**核心思想**：借鉴操作系统的分层存储与虚拟内存概念，提出"virtual context management"，通过在快/慢存储之间迁移信息，模拟更大的有效上下文。
**机制**：多层记忆+中断式控制流(interrupts)管理记忆读写与对话流程，使模型在固定窗口内完成长文档分析与多会话聊天。
**评测**：在文档分析与多会话聊天中验证可扩展上下文与长期交互记忆；代码与数据公开（memgpt.ai）。

#### 32. SocialMemBench: Are AI Memory Systems Ready for Social Group Settings? (arXiv:2605.17789, Owolabi, 2026)
**定位**：面向社交群体场景的记忆基准，强调"群体规范"与"个体例外"的区分、成员离群后的归因与共享历史。
**规模与构造**：5类社交原型（密友/家庭/娱乐/兴趣社群/熟人网络），3档群规模（4–30），共430 persona、7,355轮对话、1,031 QA、9类问题。
**关键发现**：Mem0/LangMem/Graphiti/Cognee等记忆框架在全43网络仅0.12–0.18，显著低于未压缩检索0.345与full-context 0.369（GPT-4o-mini），暴露"单流混叠、时间状态覆写、规模化实体合并、跨persona缺失、规范-个体混淆"五类失败模式。

#### 33. Evaluating Long-Horizon Memory for Multi-Party Collaborative Dialogues (EverMemBench) (arXiv:2602.01313, Hu et al., 2026)
**定位**：首个覆盖"多群体/多角色/跨主题"协作式长程记忆基准。
**数据特性**：多方多群对话>100万token，跨主题高密度交错、决策随时间演化；2400 QA覆盖细粒度召回、记忆意识、用户画像理解三维度。
**结论**：多跳归因在oracle证据下仍仅26%准确率；时间推理在缺少显式"版本语义"时系统性失败；相似度检索对隐式相关信息召回不足。

#### 34. MemGym: a Long-Horizon Memory Environment for LLM Agents (arXiv:2605.20833, Xu et al., 2026)
**定位**：面向agentic执行的长期记忆环境，强调"memory-isolated"评分以解耦推理/工具使用与记忆能力。
**评测轨道**：工具对话(tau2-bench)、深度检索(MEMGYM-DR)、编码(SWE-Gym/MEMGYM-CODEQA)、电脑使用(WebArena-Infinity)。
**关键贡献**：提出轻量MemRM奖励模型（Qwen3-1.7B QLoRA微调）评估压缩质量，避免高成本完整回放。

#### 35. ExpGraph: Model-Agnostic Experience Learning with Graph-Structured Memory for LLM Agents (arXiv:2605.30712, Feng et al., 2026)
**方法**：将历史轨迹总结为可复用技能/失败经验节点，构建自演化经验图；通过图扩散与效用排序检索经验。
**RL训练**：检索copilot用"有/无经验"性能对比反馈进行强化学习；经验图在线更新。
**结果**：在ExpSuite与ALFWorld/AppWorld等多步环境上提升显著，同时降低交互步数。

## Recommendations
1. **若要构建生产级对话记忆系统**：优先采用Mem0式"抽取-整合"两阶段pipeline作为基线（延迟低、易部署，相对OpenAI Memory有26% LLM-as-Judge相对提升、91%延迟降低），若需关系推理再上Mem0g图记忆(Neo4j)；阈值标准：当LOCOMO/LongMemEval上knowledge-update类准确率<40%时，应引入Memory-R1式RL训练的Manager学习UPDATE而非DELETE+ADD。
2. **若研究方向是长上下文/长程agent**：采用MEM1/MemAgent式上下文常量化（internal state覆写+Multi-Conv DAPO/GRPO）；基准线为"32K训练外推到512K RULER达95%+"，若达不到说明压缩策略未学好。
3. **若做多角色/群聊记忆（最大空白）**：用GroupMemBench + SocialMemBench + EverMemBench做主评测，目标先超过BM25/未压缩检索基线，重点攻克knowledge update、term ambiguity、角色归因与群体规范/例外分离；数据建模须保留per-speaker信念与受众标签，不可拼接为一对一。
4. **评测设计**：长上下文事实检索用LongBench/InfiniteBench的exact-match/ROUGE；交互式记忆与个性化用LongMemEval/MultiChallenge/PERSONAMEM的LLM-as-judge+实例级rubric；判官选gpt-4o并做position-swap去偏。
5. **自演化方向**：ReasoningBank（策略级记忆，存title/description/content，在WebArena/Mind2Web/SWE-Bench-Verified上有效性相对+34.2%、步数−16%）+ MaTTS优于存原始轨迹；从失败中蒸馏反事实是关键增益来源。
6. **触发重评的阈值**：当任一新记忆系统在GroupMemBench平均准确率突破60%、在SocialMemBench超过未压缩检索基线0.345、或在LongMemEval_M(1.5M token)上不掉30%时，应重新评估架构选型。
7. **面向agentic记忆训练/评测**：优先引入MemGym做memory-isolated评估与奖励建模，避免把推理/工具能力误当作记忆提升。

## 方向建议与可选idea
1. **群体规范-个体例外双层记忆**：将共享规范与个体例外分层存储，并为每条事实维护时间版本；更新策略用RL学习UPDATE而非DELETE+ADD，评测覆盖SocialMemBench与EverMemBench。
2. **角色条件化检索与受众建模**：针对提问者/目标受众做query rewriting与检索重排序，显式建模speaker-grounded信念；重点缓解term ambiguity与跨persona混淆。
3. **线程/频道级记忆路由**：对群聊子主题做thread-aware分块与门控，减少跨主题污染；在EverMemBench的多群多主题交错场景验证。
4. **记忆一致性审计器**：引入审计agent检测冲突与时间版本错配，触发回滚或版本升级；以knowledge update与冲突消解为主要reward。
5. **MemRM式记忆质量奖励**：仿MemGym的轻量奖励模型，针对"群聊记忆压缩质量"训练，降低RL训练成本；与Multi-Conv DAPO/GRPO结合。
6. **经验图记忆+群聊迁移**：将ExpGraph式经验图用于群聊记忆策略复用，探索"单群体学到的记忆规则"如何迁移到新群体。

## Caveats
- **arXiv编号较新**：GroupMemBench(2605.14498)、SocialMemBench(2605.17789)、EverMemBench(2602.01313)、MemGym(2605.20833)、AgeMem(2601.01885)、动态压缩记忆(2602.08382)均为2026新作，部分技术细节可能仅来自摘要/HTML，仍需对PDF与附录核对。
- **SHARE的会议归属**：部分二手源误标ACL 2024，正确为ACL 2025 Long Paper(2025.acl-long.704)，arXiv原始版2410.20682为2024年10月。
- **LOCOMO统计数字有版本差异**：300轮/9K token/35会话(arXiv)与600轮/16K token/32会话(部分ACL表述)并存，因数据集版本/统计口径不同，本报告并列标注。
- **MultiChallenge分数随榜单时间变化**：论文v1(Claude 3.5 Sonnet 41.4%)与Scale在线榜单(o1 44.93%、后续GPT-5约69.6%)不同，引用时须注明日期。
- **Mem0的LOCOMO结果存争议**：社区报告改动设置后排名可变（Zep反超约24%），其91%延迟降低与26%准确率提升为论文/官方口径，须独立复核。
- **动态压缩记忆论文**已与arXiv:2602.08382对应，但超参与消融细节尚需进一步核对。
- 多篇RL论文（Mem-α、AgeMem）的精确超参（学习率、batch、GPU数、训练步数）未在公开摘要充分披露，须查阅论文附录。
- 本报告基于arXiv abstract/HTML、ACL Anthology、OpenReview、官方GitHub等一手来源，但部分论文（GroupMemBench、AgeMem、动态压缩记忆、MuPaS的损失公式细节）的图表级公式未能逐一核到PDF原图，相关schema为基于abstract与正文片段的合理推断，已在文中标注"推断"。
