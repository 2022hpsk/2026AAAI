## 1. 研究定位与核心问题

### 1.1 一句话定位
SpeakerMem-R1 是**首个用于多方对话（3+ 人）记忆管理的强化学习框架**。

### 1.2 问题为什么重要（Significance）
真实场景大量存在且现有系统集体失败：
- **企业群聊**（Slack / 飞书 / Teams）：数十人共享频道，AI 要记住"谁说了什么"。
- **在线教育**：一师多生，AI 要跟踪每个学生的理解程度。
- **医疗多学科会诊 / 多人剧本杀**：需区分不同说话人来源的信息。

**硬数据支撑**（来自我调研的三个新 benchmark）：
- SocialMemBench：现有最好的开源记忆系统（Mem0/LangMem/Graphiti/Cognee）在多方社交场景准确率只有 **0.12–0.18**——几乎"没有记忆"。
- GroupMemBench：连最朴素的 **BM25 词面匹配** 都能追平甚至超过神经记忆系统，最强系统也只有 **46%**。这说明瓶颈不是"模型不够聪明"，而是**结构性缺陷**——没有把信息正确归属到说话人。

### 1.3 空白确认（Gap）
现有方法有三个**说话人盲区（speaker-blind）的结构性缺陷**，不是调参能解决的：
1. **Action space 缺 speaker_id**：把多方发言强行合并成"user"，存储时丢失"这是谁的信息"。
2. **Reward 的 speaker 盲区**：DeltaMem 的 Levenshtein 奖励只比对整体记忆状态——把 Alice 的事存到 Bob 名下，整体内容对但归属错，却仍得高分。
3. **Credit assignment 的 speaker 盲区**：Memory-R2 的 LoGo 只保证"相同起始状态"，但多方场景下不同 rollout 面对的说话人分布不同，奖励本质不可比。

## 2. 文献调研总结（共 35 篇，均有 10KB+ 中文详解）

每篇都按统一风格逐句拆解技术摘要，并标注"对本研究的可借鉴点与 delta"。按主题分组如下：

### 2.1 RL 记忆方法（奠基 + 2026 最新竞品）— 本研究的直接技术来源
| 论文 | 时间 | 核心技术 | 设定 | 对我们的价值 |
|------|------|---------|------|------------|
| **Memory-R1** | 2025.08 | 首个 RL 记忆，2-agent（Manager+Answer），仅 152 条训练即 +48% | dyadic | 证明小样本可行；架构蓝本 |
| **AgeMem** | 2026.01 | step-wise GRPO，6 类工具动作统一 LTM/STM | dyadic | 借鉴 step-wise 颗粒度 |
| **Mem-α** | 2026.01 | core/episodic/semantic 三组件，30k 训练泛化到 400k | dyadic | 记忆分层思想 |
| **MEM1 / MemSearcher** | 2025 | masked trajectory / multi-context GRPO（3B 超 7B） | dyadic | 多上下文轨迹处理 |
| **DeltaMem** ⭐ | 2026.04 | Memory-based Levenshtein **稠密过程奖励** | dyadic | **SpeakerLevenshtein 的直接前身** |
| **Memory-R2** ⭐ | 2026.05 | **LoGo-GRPO**（local rerollout 恢复公平比较）+ 渐进课程 | dyadic | **SC-LoGo 的直接前身**；消融揭示 Fact Extraction 最关键 |
| **CoMAM** ⭐ | 2026.03 | joint 端到端多 agent RL，rank-consistency 自适应 credit | dyadic | joint training 思路；+8.5–16.7% |
| **Mem-T** | 2026.01 | Memory Operation Tree（MoT-GRPO），树搜索稠密化 | dyadic | 稀疏奖励第三解法 |
| **MemBuilder** | 2026.01 | attributed dense reward + 四维记忆 | dyadic | 技术动机最接近，主要对比方法 |
| **R²-Mem** | 2026.05 | 反思式记忆搜索，F1 +22.6% | dyadic | 不涉多方，互补 |
| **DualMem / DeferMem** | 2026.05 | facts→insights 双流 / query-time 证据蒸馏 | dyadic | 记忆组织 + 推理时机 |

> ⭐ = SpeakerMem-R1 三大贡献分别"扩展"自这三篇，是 Related Work 的核心对比锚点。

### 2.2 多方对话记忆系统（均为 training-free / 无 RL）
| 论文 | 核心 | 关键局限（=我们的机会） |
|------|------|----------------------|
| **G-Memory** | 三层图记忆（insight/query/interaction），多 agent，+20.89% | training-free，读写策略不可学习；且面向 AI-to-AI 而非人类群聊 |
| **Collaborative Memory** | 共享/私有记忆 + bipartite 访问控制 + provenance | 写入是**固定规则**，不可学习 |
| **SA-LLM** | 说话人感知对比学习（多方对话生成） | 针对生成而非记忆管理 |
| **MuPaS** | role-masked SFT 训练多方对话模型 | 我们**直接借用**其 role-mask 用于 Stage 1 |
| **SHARE / MOOM** | 从电影剧本构造含共享记忆的多说话人数据集 | 数据构造借鉴价值 |

### 2.3 长程 RL 的信用分配（方法工具箱）
- **GRPO**（基线）/ **HiPER**（分层 RL + HAE，ALFWorld 97.4%）/ **HCAPO**（LLM 作 post-hoc critic 做 hindsight 信用分配）。

### 2.4 安全/隐私视角（升级 motivation）
- **MemorySafety**（Remembering More + SSGM + Privacy Risks）+ **MemFail**（5 类失败模式 benchmark）：把论文从"纯技术创新"升级为"技术 + 社会价值"双重贡献，并衍生跨说话人隐私泄露这一新评测维度。

### 2.5 关键调研结论
1. **多方 + RL 记忆 = 零篇**（4 轮 WebSearch 验证，截至 2026/6/2 仍空白）。
2. dyadic 赛道 2026 H1 出了 3 个强竞品，方法必须吸收其最新技术（已吸收）。
3. 三个竞品（CoMAM/DeltaMem/Memory-R2）**均无公开代码**——baseline 需自己复现，但 reviewer 也更难质疑。

## 3. 数据集调研

### 3.1 三个多方 benchmark（本研究的主战场，全部 2026 年新出）
| Benchmark | 时间 | 规模 | 评测维度 | 关键基准数字 |
|-----------|------|------|---------|------------|
| **GroupMemBench** | 2026.05 | 图驱动合成多方对话 | group dynamics / speaker-grounded belief / audience-adapted language | 最强系统仅 **46%**；BM25 追平神经系统 |
| **EverMemBench** | 2026.02 | **每实例 1M+ token**，多群组并行，2,400 QA | fine-grained recall / memory awareness / user profile | Oracle 多跳归因仅 ~26% |
| **SocialMemBench** | 2026.05 | 430 personas / 1,031 QA / 348 sessions，5 类社交群组原型 | 5 种失败模式诊断 | 开源框架 **0.12–0.18**；不压缩检索 0.345；目标超 0.35 |

### 3.2 dyadic 对照 benchmark（证明方法不退化）
| Benchmark | 规模 | 用途 |
|-----------|------|------|
| **LOCOMO** | 50 对话 × ~300 turn × ~9k token | dyadic 事实标准，几乎所有记忆论文都报；证明 N=1 时不退化 |
| **LongMemEval** | 500 条人工问题，5 大记忆能力 | 补"助手回复 recall + 知识更新" |
| **PersonaMem-v2** | 大规模 persona | 补"隐式个性化"；2k token 记忆 ≈ 32k context（16× 效率） |

### 3.3 关键发现：三个目标 benchmark 都没有训练集 ⚠️
- 三者**均为纯评测集**，且即便有对话也只有 QA pairs，**没有 memory 操作序列标注**（每个 turn 该执行哪个 WRITE/UPDATE/DELETE）。
- **影响**：SFT/RL 训练数据必须**完全合成**。我已设计图驱动合成 pipeline（参考 GroupMemBench 的 graph-grounded 思路 + Memory-R1 的 152 条拐点），目标先合成 200 条（50×4 场景）。

### 3.4 SocialMemBench 的 5 种失败模式（现成的 error analysis 框架）
直接用作论文错误分析章节结构，也对应我们的 5 层记忆设计：
1. **Single-stream conflation**（单流混淆：多人发言压成一条，丢归属）
2. **Temporal-state overwrite**（时序覆盖：UPDATE destructive 丢演化）
3. **Entity merging at scale**（大群组实体合并错误）
4. **Missing cross-persona knowledge**（跨 persona 知识缺失）
5. **Norm-individual conflation**（群体规范 vs 个体偏好混淆）

## 4. 当前方法设计（SpeakerMem-R1）

### 4.1 三大技术贡献
| 贡献 | 一句话 | 扩展自 |
|------|--------|--------|
| **① SpeakerLevenshtein 奖励** | per-speaker 分桶的 Levenshtein 稠密奖励，区分"归因错误"与"事实错误"，含 worst-speaker bonus 防止忽略最难说话人 | DeltaMem 的 flat Levenshtein |
| **② Speaker-Conditioned LoGo-GRPO** | local rerollout 按 active speaker set 分层，保证组内比较"相同说话人分布" | Memory-R2 的 LoGo |
| **③ 5 层 Speaker-Indexed Memory** | per-speaker（core/episodic/profile）+ group（interaction/insight），直接对应 SocialMemBench 5 类失败模式 | 平铺 fact store → 层次化 |

### 4.2 三阶段训练
- **Stage 1（SFT 热启动）**：role-masked SFT（借 MuPaS），用合成数据，验证 speaker attribution F1 > 50%。
- **Stage 2（Joint RL）**：SpeakerMem-GRPO，二维课程 **说话人数 K=3→5→8 × session 长度 8→16→32**。
- **Stage 3（端到端）**：Construction + Retrieval + Answer 联合优化。

### 4.3 与现有工作的关系（一句话总结）
> 每个技术贡献都能找到对应的 dyadic 前置工作做对比，差异清晰——这非常利于写 Related Work 和回应"这不就是加个 speaker_id 吗"的质疑。
