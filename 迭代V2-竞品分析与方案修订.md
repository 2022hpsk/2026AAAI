## Part 1：本轮新发现的关键论文（2026 年 4-5 月之前未在原详解覆盖）

### 1.1 [关键竞品] Memory-R2（arxiv 2605.21768，2026/5 月）

**全名**：Memory-R2: Fair Credit Assignment for Long-Horizon Memory-Augmented LLM Agents

**核心创新**：LoGo-GRPO 算法 = Local + Global GRPO 混合

- **问题诊断**：不同 rollout 对 memory 修改不同 → intermediate memory state 已经发散 → 同一 prompt 下 G 条 trajectory 不再"可比"，违背 GRPO 的 group-relative 假设
- **方案**：双分支
  - **Global**：trajectory-level reward，端到端长程学习（保持 GRPO 框架）
  - **Local**：从 **shared intermediate memory state** 出发，重 rollout，做 session-level 公平比较
- **效果**：F1 从 Memory-R1 的 43.14 → 50.60（LongMemEval/MSC-Self-Instruct/MemBench）

**对我们的冲击**：
1. **Memory-R2 已经做了"更公平的 GRPO credit assignment"**，但只在 dyadic LoCoMo 上做
2. 我们必须在写作时 cite Memory-R2 为最新 dyadic baseline，并强调多方场景下 memory state 发散问题更严重（speaker 多 → 分支更多）
3. **可借鉴**：LoGo-GRPO 的 local rerollout 思想，扩展为 **speaker-conditioned local rerollout**——同一 speaker 集合下做 local 比较

**其与 AgeMem 区别**：
- AgeMem step-wise GRPO：在单条 trajectory 内分 step 给 credit
- Memory-R2 LoGo-GRPO：跨 trajectory 在 shared memory state 处分支 rerollout
- 两者正交可结合

### 1.2 [关键竞品] CoMAM（arxiv 2603.12631，2026/3 月）

**全名**：Collaborative Multi-Agent Optimization for Personalized Memory System

**核心创新**：joint end-to-end RL for memory pipeline（construction + retrieval 多 agent 联合训练，自适应 credit assignment）

- **问题诊断**：Memory-R1 把 construction agent 和 retrieval agent **独立训**，错过了协同优化机会
- **方案**：MDP 形式化整个多 agent pipeline + GRPO 联合训练
- **效果**：在 PersonaMem 上比 Memory-R1 提升 8.5-16.7%

**对我们的冲击**：
1. CoMAM 证明 **joint multi-agent RL 比 sequential 更优**——我们的 SpeakerMem-R1 必须采用 joint，否则会被 reviewer 质疑
2. **可借鉴**：adaptive credit modeling，把"哪个 agent 该被 reward"做成 learnable
3. 但 CoMAM 是 dyadic，没考虑 speaker，没考虑 audience

**其局限（被作者承认）**：
- 只 3 个 agent
- cross-agent credit modeling 简化
- 没考虑隐私

**这些局限恰好是我们的差异化机会**——我们做 multi-party + speaker-aware credit modeling + 隐私层。

### 1.3 [关键竞品] DeltaMem（arxiv 2604.01560，2026/4 月）

**全名**：DeltaMem: Towards Agentic Memory Management via Reinforcement Learning

**核心创新**：Memory-based Levenshtein Distance 作为 dense process reward

**详细公式**（重要！原详解中 DeltaMem 是"批判稀疏奖励"的存在，现在我们知道它的具体方案）：

```
R_total = 0.1·R_format + 0.1·R_retrieval + 0.8·R_state_transition

R_state_transition：
  Δ_pred = predicted_memory_state - prev_state（增量集合）
  Δ_target = ground_truth_state - prev_state
  
  step1: 用 embedding similarity 算 Δ_pred 和 Δ_target 之间的 similarity matrix
  step2: 用 optimal transport 做 one-to-one matching
  step3: 过滤 similarity < τ 的对
  step4: 加 local lexical fidelity——检查 predicted 是否包含 target 的 keywords
  step5: 合成 soft-precision / soft-recall → F1
```

**效果**：DeltaMem-8B-RL 在 LoCoMo / PersonaMem / HaluMem 上显著超 Memory-R1

**对我们的冲击**：
1. **DeltaMem 已经解决"稀疏奖励"问题了**——这意味着我们原 idea 1 motivation "DeltaMem 揭示问题" 必须改写为 "DeltaMem 提出了 Levenshtein 解法但只针对 dyadic，多方场景需要 speaker-aware 状态差"
2. **可借鉴**：state-diff 作为 dense reward 的范式
3. **可扩展**：speaker-aware Levenshtein——把 memory state 按 speaker 分桶，对每 speaker 单独算 Δ

**这是 SpeakerMem-R1 v2 最大的技术升级点**。

### 1.4 [关键参考] What Training Data Teaches RL Memory Agents（arxiv 2605.23067，2026/5 月）

**核心发现**：训练数据 composition（curriculum）对 RL memory agent 的 skill 学习有放大效应

- 用 Qwen-2.5-7B LoRA + GRPO 训 Answer Agent
- **重要工程教训**：binary EM reward 在 G=4 时 gradient 为 0，必须用 token-level F1 reward
- 60-150 训练样本是 "specialization vs aggregate improvement" 的拐点
- 混合 curriculum > 单 benchmark

**对我们的冲击**：
1. **训练数据规模指引**：至少 150 条多方对话训练样本（mixed curriculum）
2. **奖励工程**：避免 binary，用 token-level F1 或更细
3. 我们可以**自己设计多方 curriculum**（GroupMemBench + EverMemBench mix）作为方法学贡献的一部分

### 1.5 [关键基准信号] GroupMemBench 真实数字

新发现：BM25 在 GroupMemBench 上**匹配或超过大多数 agent memory 系统**。state-update 类 27.1%，term-ambiguity 类 37.7%。

**对我们的冲击**：
1. **基线很低，提升空间很大** —— 多方记忆是真的难
2. BM25 强意味着 dense retrieval 不够好 →我们的 RL 训练必须打败 BM25
3. **重要观点**：可能 retrieval 本身已经接近上限，真正瓶颈是 **memory writing/organization** —— 这正好是 RL 训练的甜点

### 1.6 [新出基准] MemoryArena（arxiv 2602.16313，2026/2 月）

- benchmark，多 session interdependent agentic tasks
- 单用户为主
- 可作为我们的 ablation/transfer 维度

### 1.7 [新出基准] EvolMem（arxiv 2601.03543，2026/1 月）

- 认知驱动多 session benchmark
- 用 topic-initiated + narrative-inspired 合成
- 单用户

### 1.8 [写作弹药] ICLR 2026 MemAgents Workshop

- 跨 RL + memory + neuroscience 社区
- 我们的工作正好命中其主题，可在 paper 中 cite 作为社区共识"memory + RL 是热门方向"

## Part 3：SpeakerMem-R1 v2 设计修订

### 3.1 v1 → v2 的核心修订

| 维度 | v1 | v2 | 修订原因 |
|------|----|----|----------|
| 主算法 | step-wise GRPO（AgeMem） | step-wise GRPO + **LoGo speaker-conditioned local rerollout** | Memory-R2 已证明 local rerollout 必要 |
| 奖励主体 | R_task + R_attr + R_compr + R_aud + R_RIF | 上述 + **R_state_diff（speaker-aware Levenshtein）** | DeltaMem 证明 dense state reward 比 outcome reward 收敛快 |
| 训练 pipeline | 3-stage （SFT → LTM RL → 全 RL） | 3-stage but stage 2 用 **joint** multi-agent RL（不是 sequential） | CoMAM 证明 joint > sequential |
| 训练数据 | GroupMemBench train + 自合成 | + **mixed curriculum**（GroupMemBench + EverMemBench + SocialMemBench 训练子集） | Curriculum Study 证明 mix 效果好 |
| 奖励信号 | binary EM as fallback | **token-level F1 + LLM-judge ensemble** | Curriculum Study 证明 binary 在 G=4 时 gradient=0 |

### 3.2 v2 完整方法 spec

```python
# === SpeakerMem-R1 v2 ===

class SpeakerAwareMemory:
    """5-layer memory: per-speaker × {core, episodic, shared profile}, group × {interaction, insight}"""
    layers = {
        'per_speaker_core': dict[speaker_id, list],
        'per_speaker_episodic': dict[speaker_id, list],
        'per_speaker_profile': dict[speaker_id, list],  # PersonaMem-v2 style
        'group_interaction': list,
        'group_insight': list,
    }

class ActionSpace:
    """每个 action 带 speaker_id 和 audience_set 参数"""
    actions = [
        WRITE(content, speaker_id, audience_set, layer),
        UPDATE(fragment_id, new_content),
        DELETE(fragment_id),
        SUMMARY(fragment_ids, layer),
        PROMOTE(fragment_id, from_layer, to_layer),  # episodic → insight
        SUPPRESS(fragment_id, strength),             # MOOM forgetting
        READ_CROSS(asker_id, target_speaker_id),     # access-controlled
        NOOP,
    ]

# === 奖励 ===

def reward(traj, ground_truth):
    R_task = LLM_judge(answer, ground_truth)                  # outcome
    R_attr = speaker_attribution_F1(traj.memory, gt.speakers) # NEW (key)
    R_aud  = audience_adaptation_score(traj.outputs, gt.audience_profiles) # NEW
    R_compr = compression_ratio(traj.memory)                  # Mem-α style
    R_RIF  = forgetting_appropriateness(traj.suppress_ops, gt) # MOOM
    
    # NEW: speaker-aware Levenshtein state diff (DeltaMem 升级)
    R_state = sum(
        levenshtein_F1(traj.memory[s], gt.memory[s]) 
        for s in speakers
    ) / len(speakers)
    
    return (R_task 
            + 0.3 * R_attr 
            + 0.2 * R_aud 
            + 0.1 * R_compr 
            + 0.1 * R_RIF 
            + 0.6 * R_state)  # 主导信号（DeltaMem 经验：state 权重大）

# === 训练 ===

def train_speakermem_r1_v2():
    # Stage 1: MuPaS-style role-masked SFT
    sft_stage(data=mixed_curriculum,
              loss=role_masked_CE,
              roles=['user', 'agent_per_speaker'])
    
    # Stage 2: JOINT multi-agent RL (CoMAM inspired)
    # 同时训 construction + retrieval + write_policy 三个角色的同一模型
    joint_rl_stage(
        algorithm=StepWiseGRPO_with_LoGo,
        reward=reward,
        rollout=async_with_KL_reg,  # WebAgent-R1 防 Echo Trap
    )
    
    # Stage 3: end-to-end refinement on full multi-party task
    final_rl_stage(
        algorithm=MultiContextGRPO,  # MemSearcher
        reward=reward,
        eval=[GroupMemBench, EverMemBench, SocialMemBench],
    )

# === LoGo speaker-conditioned local rerollout (Memory-R2 升级) ===

def speaker_logo_grpo_step():
    """从 shared intermediate memory state 出发，
       但额外按 speaker_set 条件分支——
       同一 speaker 集合下的 trajectory 才在 local group 内比较"""
    intermediate_state = sample_shared_state()
    for speaker_set in stratified_speaker_partitions():
        local_rollouts = G_rerollouts(intermediate_state, speaker_set)
        local_advantage = group_relative(local_rollouts)
        # local + global 加权
        loss = global_loss(traj) + λ * local_loss(local_rollouts)
```

### 5.1 [新增 Idea 7] SpeakerLevenshtein：speaker-aware dense memory reward

**核心观察**：DeltaMem 的 Memory-based Levenshtein 是 dyadic 场景下的杰作。多方场景下，"哪个 fact 属于哪个 speaker" 是核心难点（GroupMemBench attribution 失败率高）。把 Levenshtein 扩展到 **per-speaker bucketed state diff**，本身就是一个强 contribution。

**具体方法**：
```
R_state_per_speaker = mean_s∈Speakers [
    Levenshtein_F1(memory_state[s], ground_truth_state[s])
]
```

加上一个 **cross-speaker leakage penalty**：
```
R_leak = - sum_s,t∈Speakers, s≠t [
    contamination(memory[s], facts_of(t))
]
```

**与 Idea 1 关系**：可以**作为 SpeakerMem-R1 v2 的核心奖励组件单独 ablation**，或作为短篇 paper 独立投递（EMNLP findings）。

**Selling point**：第一篇把 "memory state alignment reward" 扩展到 speaker-aware 的工作，技术 delta 清晰可量化。
