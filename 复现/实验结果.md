## 7. A-MEM / HippoRAG 全量成本（★ 实测 token，非估算；deepseek：入¥1/M·出¥2/M·缓存命中¥0.02/M·并发上限2500）

### 7.1 为什么 A-MEM 贵 + 实测 token
A-MEM 数据流：**每条消息** → `add_note` = analyze_content(生成关键词/上下文，1次LLM) + 演化(KNN找邻居+决定链接)。
实测 **~1 次 LLM 调用/消息**（不是之前估的 2）。根因=**逐条消息调用**：12万消息=12万次调用。

实测每 note token（前6条 vs 稳态30条）：

| 模式 | 调用/note | 入/出 token (稳态) | 每note成本 |
|---|---|---|---|
| 思考 deepseek-v4-flash | ~1 | ~860 / **1466** | ¥0.0038 |
| **非思考 deepseek-chat（已设为默认）** | ~1 | ~1215 / **710** | **¥0.0026** |

**关键优化**：analyze/演化是结构化输出、**不需推理**→用 `deepseek-chat`(非思考)，**输出 token ↓4×、速度 ↓4×、成本 ↓~2.4×**，不损质量。已设 `run_amem.py --amem-llm deepseek-chat`(默认)。

A-MEM 全量（**非思考、非截断、实测**；per-note 成本随消息长度变：社交短/企业长）：

| 数据集 | 消息数 | 每note(实测) | 入库成本 | 墙钟(16并发) |
|---|---|---|---|---|
| SocialMem(短) | 7,355 | ¥0.00126(实测302条网络) | **~¥9** | ~6min |
| EverMem(中) | 51,023 | ~¥0.0018(插值) | ~¥92 | ~7h |
| GroupMem(长) | 120,000 | ¥0.0026(实测30条Finance) | ~¥316 | ~16h |
| **合计** | | | **~¥417** | |
（实测：302条社交网络入库 ¥0.38、1.0调用/note、入765/出248 token。思考模式约 2.4× 更贵。我最初估的 ¥980 偏高——错在假设 2 次调用/note + 思考模式。）
→ **持久化后只需付一次**(`.amem_store/` 落盘，复跑跳过)。GroupMem 仍是大头，可 `--max-messages` 截断进一步降。
**并发说明**：批量入库已禁用周期 `consolidate_memories`(`evo_threshold=1e9`)避免"dict changed size"竞态；add_note 直接写 chroma、图存 self.memories，不影响检索。

### 7.2 为什么 HippoRAG 便宜
HippoRAG 数据流：**每 passage(=8条消息打包)** → NER(抽实体) + OpenIE(抽三元组)，2次LLM；建知识图谱。
**检索阶段完全不调 LLM**（query→抽实体→Personalized PageRank 排序）。
→ 调用单位是 passage(消息÷8)，比 A-MEM 逐条消息**少 ~4×**。

实测(e2e 6 passage，思考模式)：NER 出~213/passage、三元组 出~1369/passage、入~810 → **¥0.004/passage**。

| 数据集 | passage(÷8) | 索引成本(思考) | 索引成本(非思考≈÷2.4) |
|---|---|---|---|
| SocialMem | ~920 | ~¥3.7 | ~¥1.5 |
| EverMem | ~6,400 | ~¥26 | ~¥11 |
| GroupMem | ~15,000 | ~¥60 | ~¥25 |
| **合计** | | **~¥90** | **~¥38** |
→ HippoRAG 同样可切非思考(`--openie-llm deepseek-chat`)再省一半。

### 7.3 总对比（全量三数据集，非思考优化后）
| 系统 | 调用单位 | 全量成本 |
|---|---|---|
| BM25 | 无入库 | ~¥8 |
| Mem0 | 每批(100条) | ~¥35 |
| HippoRAG | 每passage(8条) | ~¥38 |
| **A-MEM** | **每条消息** | **~¥464**(截断后~¥30) |
**结论**：A-MEM 贵的本质是"逐条消息 1 次 LLM"；HippoRAG 靠"8条打包成1 passage + 检索不调LLM"便宜 ~12×。非思考模式对所有 LLM-heavy 系统都省 ~2.4×。
