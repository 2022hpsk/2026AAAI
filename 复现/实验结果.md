## 7. A-MEM / HippoRAG 全量成本估算（deepseek-v4-flash：入¥1/M·出¥2/M·缓存命中¥0.02/M·并发上限2500）

**A-MEM**：每条消息=1条note，每note≈2次推理LLM调用(analyze_content 生成元数据 + 演化链接)。估算每note≈1.5k输入+2k输出≈**¥0.0055/note**；批量并发16时墙钟≈~1s/note(并发只省时间、不省token)。

| 数据集 | 消息(note)数 | 入库调用 | 墙钟@16并发 | 入库成本 | +QA成本 |
|---|---|---|---|---|---|
| SocialMem | 7,355 | ~14.7k | ~2h | ~¥40 | +¥2 |
| EverMem | 51,023 | ~102k | ~14h | ~¥280 | +¥4 |
| GroupMem | 120,000 | ~240k | ~33h | ~¥660 | +¥1.5 |
| **合计** | | | **~49h** | **~¥980** | |
→ **A-MEM 全量极贵(~¥980/~49h)**：每条消息 2 次推理调用是结构性瓶颈，批量只省时间不省 token。**强烈建议 `--max-messages` 截断**(如每单元≤2000条，成本降到 ~¥50)。

**HippoRAG**：对话按 8 条/passage 分块，每passage=2次LLM(NER+三元组抽取)。e2e 实测每passage≈810输入+1582输出≈**¥0.004/passage**。

| 数据集 | passage(÷8) | 索引调用 | 索引成本 | +QA成本 |
|---|---|---|---|---|
| SocialMem | ~920 | ~1.8k | ~¥3.7 | +¥2 |
| EverMem | ~6,400 | ~12.8k | ~¥26 | +¥4 |
| GroupMem | ~15,000 | ~30k | ~¥60 | +¥1.5 |
| **合计** | | | **~¥90** | +¥8 |
→ **HippoRAG 全量可行(~¥90)**：passage 数=消息÷8 + 每passage仅2次调用，比 A-MEM 便宜约 10×。墙钟取决于 HippoRAG 内部并发(索引含 PageRank 图构建)。

**对比**：全量三数据集，BM25≈¥8 / Mem0≈¥35 / HippoRAG≈¥90 / **A-MEM≈¥980(需截断)**。
