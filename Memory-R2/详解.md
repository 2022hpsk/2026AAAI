# Memory-R2 详解

> 论文：Memory-R2: Fair Credit Assignment for Long-Horizon Memory-Augmented LLM Agents
> arxiv：2605.21768，2026 年 5 月
> 在本工作中的角色：**SpeakerMem-R1 v2 的算法借鉴对象 —— LoGo-GRPO 公平 credit assignment**

---

## 1. 它在解决什么问题

### 1.1 一句话

Memory-R2 揭露了一个被忽视的 RL 训练隐患：**当不同 rollout 对 memory 状态做了不同修改时，它们的 intermediate state 已经发散，GRPO 的 "group-relative" 假设被违背了**。然后提出 LoGo-GRPO（Local + Global）来修复。

### 1.2 关键问题：Group-relative 的隐藏假设

回顾 GRPO 的核心：

```
G 条 rollouts 共享同一 prompt → 它们的 reward 可比 → 算 group-relative advantage
```

但 memory agent 的训练里，每条 rollout 都会**修改 memory**。在 multi-session / multi-turn 设定下：
- 第 1 个 session 之后，G 条 trajectory 已经有了 **G 个不同的 memory 状态**
- 第 2 个 session 起，它们处理的"prompt 上下文"其实不一样了
- → "group-relative" 比较的是不可比对象

Memory-R2 把这个现象叫 **"divergence contamination"**——状态发散污染了 advantage 估计。

### 1.3 它对我们意味着什么

我们 SpeakerMem-R1 涉及更长的 trajectory（多方对话 + 多 turn），状态发散问题**更严重**。我们 v2 必须借鉴 Memory-R2 的 LoGo 思路，并且**多方场景下要扩展为 speaker-conditioned LoGo**。

## 2. 技术摘要：LoGo-GRPO 的双分支结构

### 2.1 核心思想

**Global branch**：保留 GRPO 的标准 trajectory-level reward，从开始到结束的整体 RL（保住长程学习能力）

**Local branch**：在 trajectory 的某些"shared intermediate memory state"处停下，然后**从这个 state rerollout G 次**——这 G 个 local rollout 共享同一 intermediate state，因此真的可比，做 session-level 的 group-relative 学习

### 2.2 算法流程

```python
def LoGo_GRPO_step(policy, batch):
    # === Global branch ===
    global_trajs = sample_G_full_trajectories(policy, batch.prompt)
    global_rewards = trajectory_reward(global_trajs)
    global_advantages = group_relative(global_rewards)
    L_global = grpo_loss(global_trajs, global_advantages)
    
    # === Local branch ===
    # 在每条 global traj 的中间，选一个 memory state 作为"shared state"
    for traj in global_trajs:
        for checkpoint in traj.intermediate_states:
            # 从 checkpoint 出发 rerollout
            local_rollouts = sample_G_from_state(policy, checkpoint, traj.remaining_prompt)
            local_rewards = session_reward(local_rollouts)  # 只算这个 checkpoint 之后的 session reward
            local_advantages = group_relative(local_rewards)
            L_local += grpo_loss(local_rollouts, local_advantages)
    
    L_total = L_global + λ * L_local
    return L_total
```

### 2.3 为何"公平"

- Global branch 学：哪条整 trajectory 整体上更优 → 长程信用
- Local branch 学：在同样的 memory state 下，下一步 memory operation 哪个更好 → 局部信用

两者结合 = 既有长程也有局部，既保 GRPO 框架又解决 divergence contamination。

### 2.4 训练成本

最大的代价：local rerollout **乘 G 倍计算**。Memory-R2 论文的 appendix 应该有 wall-clock 数据（待精读）。

实际工程上通常：
- 全 trajectory G=4-8
- local rerollout G=2-4
- 每条 trajectory 选 1-3 个 checkpoint

总计算 ~2-3 倍 GRPO。

## 4. Memory-R2 的具体设定

### 4.1 模型

Qwen2.5-7B 单 backbone，role-specific prompts 同时训 fact extractor + memory manager 两个角色。这是 **parameter-efficient co-learning**——和 CoMAM 的 joint 训练思想一致。

### 4.2 Reward 

trajectory-level reward（outcome 为主），但通过 LoGo 的 local branch 把它"分摊"到 session level。

注意：和 DeltaMem 不一样，Memory-R2 没用 dense state reward。它的解法是从**算法**层面修 credit assignment，而 DeltaMem 是从**奖励信号**层面修。**两者正交**，可结合。

### 4.3 数据

只在 LoCoMo 上训（2 个 conversation 训练）。这是 Memory-R2 自己承认的局限——数据规模小。

## 5. 对 SpeakerMem-R1 v2 的具体借鉴

### 5.1 直接借鉴：speaker-conditioned LoGo

Memory-R2 是 dyadic，local rerollout 是按 **memory state 分组**。我们扩展到 multi-party：

```python
def speaker_conditioned_LoGo_GRPO(policy, batch):
    global_trajs = sample_G_full(policy, batch)
    L_global = grpo_loss(...)
    
    # local: 按 (intermediate_state, active_speaker_set) 分组
    for traj in global_trajs:
        for checkpoint in traj.intermediate_states:
            active_speakers = checkpoint.active_speakers
            
            # 关键扩展：rerollout 时固定 speaker set
            local_rollouts = sample_G_from_state_with_speakers(
                policy, checkpoint.state, active_speakers, traj.remaining
            )
            
            local_advantages = group_relative(reward(local_rollouts))
            L_local += grpo_loss(local_rollouts, local_advantages)
    
    return L_global + λ * L_local
```

**为什么 speaker-conditioned**：不同 speaker set 下的 memory operation 难度差异巨大（3 人对话 vs 7 人对话）。如果不分层，会把高难度低分 rollout 错误归因为"模型差"。

### 5.2 与 DeltaMem 的结合

我们 v2 既要 dense state reward（DeltaMem）也要 fair credit（Memory-R2）：

```python
def combined_v2_loss(policy, batch):
    # Memory-R2 LoGo 框架
    global_trajs = sample_G_full(...)
    
    # 每条 traj 上算 dense state reward (DeltaMem) per turn
    for traj in global_trajs:
        traj.per_turn_state_reward = [
            speaker_aware_levenshtein(s_pred, s_target) 
            for s_pred, s_target in zip(traj.states, batch.gt_states)
        ]
    
    # Global GRPO 用 sum of per-turn state reward
    global_advantage = group_relative([sum(t.per_turn_state_reward) for t in global_trajs])
    L_global = grpo_loss(global_trajs, global_advantage)
    
    # Local rerollout 在 checkpoint 处，用 from-this-point state reward
    for traj in global_trajs:
        for cp in traj.checkpoints:
            local_rollouts = rerollout_from(cp)
            local_rewards = [
                sum(r.per_turn_state_reward[cp.turn:]) 
                for r in local_rollouts
            ]
            local_advantage = group_relative(local_rewards)
            L_local += grpo_loss(local_rollouts, local_advantage)
    
    return L_global + λ * L_local
```

**核心 idea**：把 DeltaMem 的 dense reward 喂给 Memory-R2 的 LoGo 框架。这是 v2 算法层的关键创新——**第一次把 dense state reward 和 fair credit assignment 结合起来**。

### 5.3 checkpoint 选择策略

Memory-R2 没明说 checkpoint 怎么选。多方场景下我们可以做 **speaker-driven checkpoint selection**：

- 每当某 speaker 第一次出现 → 一个 checkpoint
- 每当 memory state 中 speaker-bucket 的 fact 数翻倍 → 一个 checkpoint
- 每当 cross-speaker reference 出现 → 一个 checkpoint

这些 checkpoint 是 **"speaker-grounded 学习困难点"**，正是我们最想给 local credit 的地方。

### 5.4 局部 reward 的 speaker-aware 加权

local rerollout 的 reward 也按 speaker 加权：

```python
def local_reward(rollout, active_speakers):
    base = speaker_aware_levenshtein(rollout.state, gt.state)
    # 只看 active speakers 那部分 reward
    return mean([base[s] for s in active_speakers])
```

## 6. Memory-R2 的局限（我们的差异化机会）

### 6.1 局限 1：dyadic

只在 LoCoMo 上验证，user-assistant 两方。

**→ 我们做 multi-party，且 LoGo 扩展为 speaker-conditioned**

### 6.2 局限 2：训练数据极小（2 条对话）

容易过拟合。Memory-R2 也承认数据 efficiency 问题。

**→ 我们用 mixed curriculum（GroupMemBench + EverMemBench + 合成）**

### 6.3 局限 3：fixed answer agent

Memory-R2 假设 downstream answer agent 是 fixed 的（用预训 7B model），只训 memory operations。

**→ 我们 v2 联合训 memory + answer + attribution policy（更接近 CoMAM 的 joint 训练）**

### 6.4 局限 4：reward 仍是 outcome

Memory-R2 只解了 credit assignment，没解 outcome reward 稀疏问题。

**→ 我们结合 DeltaMem dense reward + Memory-R2 LoGo = 双重 mitigation**

### 6.5 局限 5：curriculum tuning empirical

Memory-R2 自己说"curriculum 怎么排需要 empirical 调"。

**→ 我们引用 Curriculum Study 论文 (2605.23067) 的发现：mixed > narrow，token-F1 > binary**

## 7. 对比表：v1 / v2 中的训练算法

| 维度 | v1 (only step-wise GRPO) | v2 (LoGo + step-wise + DeltaMem reward) |
|------|--------------------------|------------------------------------------|
| Credit | step-wise 给单 trajectory 内 step credit | step-wise + LoGo 跨 trajectory checkpoint credit |
| Group-relative 假设 | 假设状态可比（隐患） | 通过 local rerollout 显式保证可比 |
| Reward | outcome heavy | dense state + outcome + speaker-aware 多项 |
| 计算成本 | 1x GRPO | ~2-3x GRPO（local rerollout） |
| 期望收敛速度 | 中 | **快**（dense reward + fair credit 双 mitigation） |

## 8. Related Work 写法

> **Credit Assignment for Memory RL.** Recent work has identified credit assignment as a central challenge for training memory agents. AgeMem [cite] introduces step-wise GRPO that distributes credit across sub-trajectory steps. Memory-R2 [cite] further observes that diverging memory states across rollouts violate the group-relative assumption of GRPO, and proposes LoGo-GRPO with local rerollouts from shared intermediate states. Both methods, however, operate exclusively in dyadic settings. Multi-party memory introduces an additional source of variability—**varying speaker sets across rollouts**—that further amplifies state divergence. Our work generalizes LoGo-GRPO to a **speaker-conditioned** form, where local rerollouts are stratified by active speaker set, combined with **DeltaMem-style dense state reward** to provide multi-grain mitigation for sparse outcome signals.

---

## 9. 整体设计哲学

Memory-R2 的核心洞察：**"Group-relative comparison demands shared starting points."**

我们 v2 的扩展：**"Shared starting points in multi-party means shared (state, speaker set)."**

这是从单一维度（state）到双维度（state × speaker set）的自然扩展。
