# SpeakerMem-R1 问题导向汇报文档（V2）

**定位**：按导师要求重组——**关键是"解决什么问题"，不是"怎么做"**。本文聚焦：问题 → 指标 → SOTA 方法 → 不足 → Related Work 逐篇拆解 → 我们的方法/指标/预期。
**日期**：2026-06-03

---

## 0. 一句话

> **问题**：多角色（3+ 人 / 群聊）对话里，AI 必须精准区分"谁说了什么、说给谁"，但**现有所有系统都做不到**——要么是 dyadic（一对一）RL 记忆，要么是 training-free 的多方系统。
> **机会**：多方 + 说话人锚定 + RL 训练，截至 2026/6 **零篇先例**。

---

## 1. 解决什么问题（关键）

**核心问题**：为多方对话设计**可学习的、说话人锚定的记忆管理**——让 agent 在群聊中正确地"把每条信息归属到正确的人、正确的时间、正确的受众"，并据此作答。

**为什么是真问题（不是造出来的）**：
- 真实刚需：企业群聊（Slack/飞书/Teams）、在线教育（一师多生）、医疗多学科会诊、客服/合规、多 agent 协作。
- 现有系统**集体失败**的硬数据：
  - **SocialMemBench**：最好的开源记忆系统（Mem0/LangMem/Graphiti/Cognee）在多方社交场景仅 **0.12–0.18**（≈"没有记忆"）；不压缩的完整检索 reference = 0.345。
  - **GroupMemBench**：最强系统仅 **46%**；连朴素 **BM25** 词面匹配都能追平甚至超过神经记忆系统 → 说明瓶颈是**结构性缺陷**（没把信息正确归属到说话人），不是模型不够聪明。
  - **EverMemBench**：即便给 Oracle 证据，多跳归因也仅 **~26%**。

**问题的本质**：现有记忆写入策略**抹掉了 speaker- 与 audience-grounded 的结构线索**。多方比 dyadic 多出三个 dyadic 中不存在的挑战：(1) 说话人归属、(2) 受众适配、(3) 跨说话人隐私。

## 2. 用什么指标衡量（有哪些指标）

| 层次 | 指标 | 说明 |
|------|------|------|
| **任务层** | QA **token-level F1**、EM | 主指标用 F1（binary EM 在 G=4 时梯度为 0） |
| **记忆质量层** | **Speaker attribution F1 / accuracy** | 多方核心：每条 fact 是否归属到正确说话人 |
| | Recall@K | 检索召回 |
| | Memory-state Levenshtein F1 | 过程指标（记忆状态与 GT 的接近度） |
| **多方专属** | per-category accuracy | SocialMemBench 5 类失败模式分项 |
| | Multi-hop attribution acc | EverMemBench 多跳归因 |
| | Audience adaptation score | 回答是否适配提问者 |
| | **Cross-speaker leakage rate** | 跨说话人隐私泄漏率（**我们新增的评测维度**） |
| **效率层** | p95 latency、token cost、context efficiency | Mem0 p95 几百 ms；2k 记忆 ≈ 32k context（16×） |
| **兼容性** | **LoCoMo F1** | 证明 N=1（dyadic）时方法不退化 |

**各 benchmark 的报告指标 + 关键数字**：
- GroupMemBench：EM / F1 / attribution acc；**最强 46%**，知识更新类 27.1%、术语歧义类 37.7%。
- EverMemBench：Recall@K / F1；Oracle 多跳归因 **~26%**。
- SocialMemBench：per-category acc；开源 **0.12–0.18**，uncompressed 0.345，Gemini full-context 上界 **0.721**，**目标 > 0.35**。
- LoCoMo：F1（dyadic 标准；Memory-R2 报 **49.67**）。

## 4. 现有方法共同的不足（= 我们的机会）

1. **所有 RL 记忆方法都是 dyadic**：动作空间、奖励、信用分配都假设"只有一个对方"，没有说话人维度。
2. **所有多方工作都是 training-free / 仅评测**：读写策略是固定规则，不可学习；或只做对话生成、只做 benchmark。
3. **三个具体的 speaker-blind 结构缺陷**（不是调参能解决的）：
   - **Action space 缺 speaker_id**：多方发言被压成"user"，存储即丢归属。
   - **Reward 的 speaker 盲区**：DeltaMem 的 flat Levenshtein 只比对整体记忆状态——把 Alice 的事存到 Bob 名下，内容对、归属错，仍得高分。
   - **Credit assignment 的 speaker 盲区**：Memory-R2 的 LoGo 只保证"相同起始状态"，但多方下不同 rollout 的 active speaker 分布不同，奖励本质不可比。
4. **三个多方 benchmark 都无训练集**：无法直接 RL，训练数据须合成。

## 5. Related Work 逐篇拆解（解决什么问题 / 动机·针对什么缺陷 / 还有什么不足）

### 5.1 RL 记忆方法 + 单用户记忆系统（按时间排序）

> 客观列出每篇"解决什么 → 怎么做 → **自身**还差什么"。这里的"不足"只讲它们各自的问题，不涉及说话人维度——说话人盲区是它们的**共同**缺口，统一放在表后总结。

| 工作 | 时间 | 解决的问题（动机，**细**） | 方法简述 | 仍然不足（**自身**的局限，细） |
|------|------|------------------|---------|----------------|
| MemoryBank | 2023 | 长期对话记忆无限增长、且没有"该忘就忘"机制；想模拟人类记忆的衰减与个性化 | 把艾宾浩斯遗忘曲线引入：每条记忆有强度 S(t)=S₀·e^(−t/τ)，被检索就重置强化、低于阈值即丢弃；并渐进维护 user profile | 纯规则、无任何学习；只有**单一** user profile（无法多用户）；公认最弱 baseline、精确事实回忆差；衰减常数 τ 全局固定、对不同信息一刀切 |
| A-MEM | 2025.02 | Mem0 等用预定义 schema 组织记忆太僵硬；想要无需 schema 的灵活关联 | 仿 Zettelkasten 卡片盒：每条记忆是"原子卡片"（一句话），卡片间由 LLM 在写入时**动态生成**链接（extends/contradicts…），检索靠链接遍历 | 链接类型与质量**完全由 LLM 启发式决定、不稳定且不可学**；整体性能略低于 Mem0（约 −1~2 点）；无写入策略学习 |
| Mem0 / Mem0g | 2025.04 | 把全部历史塞进 prompt 既贵又超窗口；想要 production 级、低延迟低成本的长期记忆 | 对每条抽取的 fact 向量检索相似旧记忆，再让 LLM 判 ADD/UPDATE/DELETE/NOOP；Mem0g 增 Neo4j 实体关系图 | 写入决策是**固定的 LLM 启发式、不可端到端学**（Memory-R1 正是把它的动作设计变成 RL 目标）；每条 fact 都要一次 LLM 调用、有延迟/成本；speaker-agnostic；Mem0g 依赖 Neo4j 部署重；多方下被 BM25 追平 |
| MemGAS | 2025.05 | 记忆粒度单一、检索权重不会自适应；想要多粒度关联 + 按需检索 | 多粒度记忆单元 + GMM 聚类建立关联 + 熵路由决定各粒度的检索权重 | 多粒度 + 聚类带来额外构建/维护开销；training-free 不学习；聚类质量依赖 embedding |
| MemoryOS | 2025.06 | 长期个性化记忆缺乏系统化的存储/调度（什么进短期、什么沉淀长期） | 借操作系统思想：短/中/长期分层 + 分页 / FIFO 式调度在层间搬运记忆 | 调度规则是**启发式、不可学**；分层容量/阈值需手工设定；本质仍单用户 |
| MemAgent | 2025（年中） | 超长文档 QA：扩窗 O(N²) 吃显存、RAG 检索有损（无法全局推理）；想要线性复杂度 | chunk 顺序读，每读一段就**覆盖式重写**一段定长（~1k token）人类可读文本记忆；用 Multi-Conv DAPO 训（K 个独立 context 共享同一 trajectory 奖励） | 定长记忆**装不下精细事实**（事实型任务不如 RAG）；每 chunk 都重写整段、输出 token 成本高；Multi-Conv 优势分配粗糙（不区分哪个 chunk 关键）|
| Memory-R1 | 2025.08 | Mem0 等写操作是固定规则、次优；**首次**想"把记忆操作本身变成可学习目标" | 双 agent：Memory Manager 学 ADD/UPDATE/DELETE/NOOP，Answer Agent 做 Memory Distillation（从 RAG 召回 60 条里蒸馏再答）；**outcome-only EM 奖励** + GRPO，仅 152 条训练 | 奖励仅最终 EM、**极稀疏**（开放生成 / 多跳就失效，被 DeltaMem 正面批评）；动作仅 4 类、UPDATE 合并语义模糊；双 agent rollout 成本翻倍；60 条候选是 hardcoded |
| MEM1 / MemSearcher | 2025 | 多上下文 / 长轨迹下记忆非线性演化、训练效率低 | masked trajectory 处理非线性轨迹 / multi-context GRPO（让 3B 超 7B baseline）| 任务相对窄；仍是单上下文 GRPO 范式的扩展；缺结构化记忆设计 |
| LightMem | 2025.10 | 检索型记忆准确率不稳、而反复大模型调用累积高延迟；想要"高效" | 在线 / 离线分离 + LLMLingua-2 / 熵压缩蒸馏原文 + 话题分段；把昂贵的 consolidation 放离线 | 压缩可能丢细节、影响精确回忆；离线 consolidation 非实时、有延迟；仍单用户 |
| GAM | 2025.11 | long-context agentic memory：写时压缩会丢信息，想"延迟到用时再深挖" | Memorizer + Researcher 双 LLM：写时只做轻量 page-store，用时 JIT 式 deep-research 检索；RL 优化 | 双 LLM 推理成本高；JIT 深挖增加查询延迟；仍单用户 |
| AgeMem | 2026.01 | 此前记忆是"外挂模块"、操作被切到外部规则；想让记忆成为策略的**一等认知动作** | 取消独立记忆管理器，主策略统一管 LTM/STM；6 类工具动作（ADD/UPDATE/DELETE/RETRIEVE/SUMMARY/FILTER）与文本生成共享解码空间；三阶段课程 + step-wise GRPO；奖励 R_task+R_context+R_memory **等权** | RETRIEVE 仍是简单向量检索；step-wise 切片**无 counterfactual**、credit 仍不够精确；三项奖励等权是工程简化、非数据驱动 |
| Mem-α | 2026.01 | 扁平记忆库检索 noise 大（核心偏好与随口事实在向量空间一样近）；想让"记忆架构本身"可学 | core/episodic/semantic 三组件（认知科学分层）+ 各自专用工具 + GENERALIZE 抽象；RL 学跨层路由；奖励 R_QA + β压缩(0.05) + γ语义一致 | 三层边界是 **hardcoded、非学出**；GENERALIZE 会放大源 episode 的误差；**只学 writing、reading 仍用固定 RAG**（无端到端） |
| Mem-T | 2026.01 | 长程稀疏奖励 + 单操作价值难估（哪一步 WRITE/DELETE 关键说不清） | Memory Operation Tree：每步展开 K 个候选操作做树搜索，把 leaf 的 outcome 反传成每个中间节点的稠密优势；再用 LLM 作 hindsight critic 反事实归因；配分层 DB | **树搜索计算开销高**（每步 K 候选展开）；hindsight 用 LLM 推理近似、有噪声；仍 dyadic |
| MemBuilder | 2026.01 | 记忆构建奖励稀疏（只 end-of-session）+ 多维记忆难归因（一条操作对哪维贡献多少不知） | ① session 级中间奖励（每 session 自动合成 QA 即时打分）② 贡献感知梯度加权（记忆被后续检索越多，其产生操作梯度权重越大）；四维记忆；4B 超多数闭源大模型 | 贡献信号用"**检索频率做代理**"、较粗（被检索多 ≠ 真有用）；冷启动需人工标注记忆操作；仍单流、speaker 混在一起 |
| CoMAM | 2026.03 | Memory-R1 把 construction/retrieval **分开 sequential 训**、错过两者耦合反馈；想端到端联合训 | 整条 pipeline 当一个 MDP，construction/profile/retrieval 三 agent 共享 backbone joint GRPO；adaptive credit = local 与 global reward 的 **Spearman 排名相关性**给每 agent 自适应权重；+8.5~16.7% | 仅 3 个 agent、设定简化；agent 间关系仅 learnable 标量权重、**未显式建模因果**；**完全不处理隐私**；仍 dyadic、outcome reward（无 dense state） |
| DeltaMem | 2026.04 | **直接批 Memory-R1**：用下游 QA 对错当奖励太稀疏，GRPO 同组可能全 0、梯度消失；想"写记忆那一刻就给奖励" | Memory-based Levenshtein：把记忆看 fact set，比对"应发生的增量 Δ_target"与"实际增量 Δ_pred"——embedding 余弦 + Hungarian 匹配 + 阈值 τ + 词面保真，得 soft-F1；R=0.1fmt+0.1ret+**0.8state** | flat、**不分说话人**（归属错与事实错同等惩罚）；需 GT memory state 标注、训练数据纯合成有 bias；是"结果端"稠密、**仍不知哪个操作关键**；记忆单调增、无遗忘；**无公开代码** |
| TreeMem | 2026.05 | 多 agent 记忆信用要么平摊（不知谁贡献）、要么需人工标 intermediate（贵）；想**无标注**推 per-agent 信用 | 建"记忆-答案依赖树"：沿"哪条记忆被检索 → 哪个操作产生它"的因果链，用互信息把最终 reward 分配到各 agent | 因果链**依赖"哪条被检索"**，检索一旦有噪声信用就带偏；互信息估计是近似；仍 dyadic、**无公开代码** |
| Memory-R2 | 2026.05 | 揭露 GRPO 隐患：不同 rollout 改了不同记忆态、中间态发散，"group-relative"假设被违背（divergence contamination） | LoGo-GRPO：global 保长程 + 从共享中间态 local rerollout 做可比的 session 级 group 比较；Qwen2.5-7B co-learning（fact extractor + manager）；3 阶段课程 8→16→32 | local rerollout 增计算（~2–3× GRPO）、只到 session 级；**消融显示 LoGo 只 +3 F1，真正命脉是 fact-extractor co-learning（单训它崩到 28.30）**；训练数据极小（2 条对话）易过拟合；answer agent 固定；**无公开代码** |
| R²-Mem | 2026.05 | 一次检索常不准、缺对检索结果的反思修正 | 反思式记忆搜索：检索 → 反思 → 再检索的迭代经验记忆搜索，LoCoMo F1 +22.6% | 偏读侧（不解决写策略）；反思迭代增加推理成本/延迟；仍 dyadic |
| DeferMem | 2026.05 | 写记忆时就精炼会过早丢信息；想"延迟到 query 时再按问题精炼" | segment-link 高召回粗粒度存储；query 时用 DistillPO（faithfulness+sufficiency−长度）从噪声候选蒸馏自包含证据 | **只做读侧、完全不管写策略**；query 蒸馏增加推理延迟；仍 dyadic |
| DualMem | 2026.05 | 单层记忆难兼顾"原始事实"与"高层洞察" | facts→insights 双流：底层存事实、上层蒸馏洞察、双流协同 | 双流构建 / 同步复杂；偏读侧组织；仍 dyadic |

> **统一总结**：以上 21 项工作**全部是 dyadic / 单用户设定**——记忆条目不绑定说话人、读写策略不区分"谁说的、说给谁"。这正是 SpeakerMem-R1 切入的空白。

### 5.5 值得先去复现的工作（优先级 + 理由）

> 判断依据三条：(1) 是否是审稿人**必然预期**的 baseline；(2) 是否**有开源代码**（决定复现成本）；(3) 是否是我们方法的**直接前身**（必须吃透才能扩展）。原则：先做"低成本高收益"的，再做"必须自实现的核心前身"。

**第一梯队 · 立刻做（低成本、验证 benchmark + 必报 baseline）**
- **Mem0**（开源、production）：是 GroupMemBench / SocialMemBench / LoCoMo 的标准 training-free baseline。跑通它 = 同时**验证我们的评测 pipeline** + 拿到 training-free 对照数字。**收益最高、成本最低，第一个做。**
- **BM25**（几行 rank-bm25）：GroupMemBench"BM25 追平神经记忆系统"是我们 motivation 的核武器，必须**自己复现这个数字**才能理直气壮地讲。

**第二梯队 · RL 训练脚手架（有开源代码可 fork）**
- **MemAgent**（有开源代码 + Multi-Conv DAPO）：它的"K 个独立 context 共享同一 trajectory 奖励"与我们"每个说话人一条独立流"**高度同构**。fork 它改成 per-speaker 多流，**比从零搭 RL 框架快得多**——是我们最省力的 RL 起点。
- （可选）**LightMem**（zjunlp 开源、ICLR'26）：高效型近期 training-free baseline，跑通成本低，顺手加一个强对照。

**第三梯队 · 我们方法的直接前身（无公开代码、须自实现，按收益排序）**
- **DeltaMem 的 Memory-based Levenshtein 奖励**（贡献①的基础）：reward 函数相对独立（embedding + Hungarian），我们已有初版。**先单独把它跑通、在合成数据上验证它能区分好/坏记忆**——这是验证 SpeakerLevenshtein 的最小单元。
- **Memory-R2 的 co-learning（先）+ LoGo（后）**（贡献②的基础）：注意它**自己的消融**——LoGo 只 +3 F1，真正命脉是 **fact-extractor co-learning（单训它崩到 28.30）**。所以**先复现 speaker-aware fact extraction 的 co-learning（高收益），LoGo 次之**。
- **Memory-R1**（RL baseline）：outcome-only 双 agent，是我们"加过程奖励"的天然对照组；152 条即可训、资源友好。

**暂不优先**：CoMAM（无代码、joint 复杂；但 adaptive credit 只是 Spearman、可后期几行加上）、Mem-T / TreeMem（树搜索、无代码、开销大）、MemoryOS / MemGAS / GAM（cite 即可，不必全复现）。

> **我的判断（供老师指正）**：先用 **Mem0 + BM25** 把 benchmark 与 motivation 数字坐实 → 再 fork **MemAgent** 拿到能训的 RL loop → 然后把 **DeltaMem 的奖励** 和 **Memory-R2 的 fact-extractor co-learning** 这两块"前身"在多方数据上自实现并验证。这条路径风险最低，且每一步都直接喂给我们的三大贡献；**关键的自己判断是：不盲目复现全部 SOTA，而是抓"必报 baseline + 有代码的脚手架 + 两个核心前身"三类。**

## 6. 我们的方法（简）+ 指标 + 预期

### 6.1 方法（一句话，"怎么做"是次要的）
**SpeakerMem-R1 = 首个多方对话 speaker-grounded RL 记忆框架**。三个技术贡献，分别针对 §4 的三个 speaker-blind 缺陷：
- **SpeakerLevenshtein 奖励**（扩展 DeltaMem）：按说话人分桶的稠密奖励，区分"归属错"与"事实错"。
- **Speaker-Conditioned LoGo-GRPO**（扩展 Memory-R2）：local rerollout 按 active speaker set 分层，保证公平比较。
- **5 层 Speaker-Indexed Memory**：per-speaker（core/episodic/profile）+ group（interaction/insight），对应 5 类失败模式。

### 6.2 我们报告的指标
主：三个多方 benchmark 的 EM/F1/attribution acc + per-category acc；
专属：cross-speaker leakage rate（新维度）、audience adaptation；
兼容：LoCoMo F1（不退化）；效率：token / context efficiency。

### 6.3 预期结果
- **SocialMemBench**：超过 uncompressed 0.345，**目标 > 0.35**（开源系统只有 0.12–0.18）。
- **GroupMemBench**：超过最强 46%，attribution 类（知识更新 27.1% / 术语歧义 37.7%）显著提升。
- **关键消融**：per-speaker vs flat Levenshtein（预期 +5–10 F1）、joint vs sequential（+4–8）、LoGo vs 无（+2–4）。
- **LoCoMo**：接近 Memory-R2 的 49.67，证明 dyadic 不退化。
- **贡献**：首个多方 RL 记忆方法 + 新评测维度（跨说话人隐私泄漏）+ 开源代码与合成数据。
