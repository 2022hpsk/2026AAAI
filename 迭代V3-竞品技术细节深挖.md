## Part 1：CoMAM 技术细节

**论文**：Collaborative Multi-Agent Optimization for Personalized Memory System  
**arXiv**：2603.12631，2026/3（最新版 2026/4/27）  
**作者**：中科大 + ByteDance Seed  
**代码**：❌ 未公开（未找到官方 repo）

### 1.1 Adaptive Credit Modeling 实现方式

**具体机制**：基于 **ranking consistency 的 per-agent adaptive scalar weight**（不是 attention-style）

```python
# CoMAM 的 adaptive credit modeling
# 核心：衡量每个 agent 的 local reward 与 global reward 的"排名一致性"

def compute_agent_credit_weight(
    local_rewards: list,   # agent 自身的 local task reward
    global_rewards: list,  # 整个 pipeline 的 global reward
) -> float:
    """
    计算 rank correlation（类 Spearman/Kendall）：
    如果 local reward 的排名顺序与 global reward 的排名顺序高度一致，
    说明这个 agent 对全局目标贡献大 → 分配更大的 α_i
    """
    rank_consistency = spearman_correlation(
        rank(local_rewards), 
        rank(global_rewards)
    )
    alpha_i = normalize(rank_consistency)  # 映射到 [0,1]
    return alpha_i

# 最终每个 agent 的优化目标
def agent_objective(local_reward, global_reward, alpha_i):
    return local_reward + alpha_i * global_reward
```

**与原 CoMAM 详解的修订**：之前猜测是"learnable weight 或 attention"，实际是基于统计排名相关性的自适应系数，**不需要额外梯度传播**，是计算出来的。

### 1.2 三类 Agent 架构

| Agent | 职责 | 输出 | Local Reward |
|-------|------|------|-------------|
| **Extraction Agent** | 细粒度 memory 构建 | 细粒度 fact entries | information coverage |
| **Profile Agent** | 粗粒度 user profile 构建 | 粗粒度 profile 摘要 | profile accuracy |
| **Retrieval Agent** | 记忆检索与问答 | 答案 | QA accuracy（global） |

**顺序式 MDP**：上游 agent 输出作为下游 agent 状态输入。整体 MDP 联合优化。

### 1.3 消融实验关键结果

| 配置 | 说明 |
|-----|------|
| CoMAM（完整自适应信用） | **最优** |
| AT-GRPO（均匀全局信用） | 次优 |
| 独立优化（无 joint） | 最差 |

**结论**：自适应信用分配机制 > joint MDP 结构 > 无协作基线

### 1.4 CoMAM 对 SpeakerMem 的含义（更新版）

**原有认知**（V2）：CoMAM 证明 joint > sequential

**新增认知**（V3）：
1. CoMAM 的 adaptive credit = **rank consistency**，实现比想象中简单
2. 可以直接借鉴：把每个 speaker 的 memory construction sub-task 看成一个"agent"，计算其 local reward 与 global QA reward 的 rank consistency → speaker-level credit
3. **多方扩展**：CoMAM 的 MDP 未建模 speaker identity，这正是我们的差异化 —— 我们扩展为 speaker-conditioned MDP

## Part 2：DeltaMem 技术细节

**论文**：DeltaMem: Towards Agentic Memory Management via Reinforcement Learning  
**arXiv**：2604.01560，2026/4/3  
**代码**：❌ "code will be open-sourced upon acceptance"  
**基座模型**：Qwen3-4B（DeltaMem-4B-RL）+ Qwen3-8B（DeltaMem-8B-RL）

### 2.1 Levenshtein 阈值 τ 的选择

**关键发现**：
- τ 作用：过滤低相似度的 memory entry 匹配对，避免错误 reward 信号
- **消融发现**：τ 过小 → 低质量匹配引入噪声 → LoCoMo 上性能下滑
- **具体数值**：论文未在公开摘要披露（需精读 Appendix 的敏感性分析表）
- 我们的方法 spec 中设 τ=0.6，这是合理估计，需根据 DeltaMem 论文的实验补充（建议 τ=0.5~0.7 范围）

### 2.2 Local Lexical Fidelity 的重要性

```
DeltaMem reward 组成：
- embedding cosine similarity（语义相似度）
- local lexical fidelity（关键词覆盖度）

消融结果：加入 local fidelity → PersonaMem 提升约 +3 F1 points

含义：仅靠语义嵌入不够，还需要词面层面的精确对齐
对我们：speaker-aware Levenshtein 也应该保留 lexical fidelity 组件
```

### 2.3 训练效果

- 训练曲线：reward 快速上升后稳定在约 **0.75**（意味着 RL 收敛稳定）
- DeltaMem-8B-RL 在 LoCoMo/HaluMem/PersonaMem 上显著超越所有 training-free baseline（Mem0/LangMem/Zep/A-Mem/Memobase 等）

### 2.4 多方场景的关键局限（DeltaMem 明确未做）

- DeltaMem 定位：单 agent，persona-centric，**单用户**
- τ 在多方场景的问题：不同说话人的语言风格差异显著，同一 τ 可能对 A 说话人过严、对 B 说话人过松
- **我们的贡献机会**：per-speaker τ calibration（基于 speaker 的历史发言统计自动调整）

```python
# SpeakerMem-R1 的改进：per-speaker τ 自适应
def get_speaker_tau(speaker_id: str, speaker_stats: dict) -> float:
    """
    基于说话人的历史发言特征，动态调整 τ
    语言风格正式/词汇精准的说话人 → 高 τ（要求精确匹配）
    语言风格随意/用词多变的说话人 → 低 τ（允许更多语义模糊匹配）
    """
    vocab_consistency = speaker_stats[speaker_id]['vocab_consistency']
    # vocab_consistency ∈ [0,1]，越高越正式/一致
    base_tau = 0.6
    return base_tau + 0.15 * (vocab_consistency - 0.5) * 2
    # 结果范围 [0.45, 0.75]
```

## Part 3：Memory-R2 技术细节

**论文**：Memory-R2: Fair Credit Assignment for Long-Horizon Memory-Augmented LLM Agents  
**arXiv**：2605.21768，2026/5/20  
**代码**：❌ 未发布（论文为 2026/5 预印本）  
**架构**：Fact Extractor + Memory Manager（共享 LLM backbone，role-specific prompt）

### 3.1 LoGo-GRPO 完整消融结果

| 配置 | F1 | B1 | 说明 |
|-----|----|----|------|
| Memory-R2（完整） | **49.67** | **43.77** | 最优 |
| 标准 GRPO（去掉 LoGo） | 46.62 | 40.97 | -3.05 F1，验证 LoGo 的贡献 |
| 仅训练 Memory Manager（无 co-learning） | 45.34 | — | -4.33 F1 |
| 仅训练 Fact Extractor | 28.30 | — | 崩塌 |

**关键洞察**：
1. **LoGo 的贡献**：约 +3 F1（有意义但不是革命性的）
2. **最大贡献**：Fact Extractor 与 Memory Manager 的 co-learning —— 去掉 co-learning 后下降 4.33 F1，去掉 Fact Extractor 直接崩塌（28.30），说明**信息抽取是整个 pipeline 的最薄弱环节**
3. **含义**：对 SpeakerMem-R1，speaker-aware 的 **fact extraction**（从多方对话中正确提取带说话人标注的事实）可能是最关键的组件

### 3.2 Curriculum 设计（三阶段进阶）

```
Stage 1: 8-session 训练数据  → 学基础 memory 操作，减少误差传播
Stage 2: 16-session 训练数据 → 引入更强的跨 session 依赖
Stage 3: 32-session 训练数据 → 完整长视野优化
```

- 我们的 curriculum 应该类似：先学 3 人对话（低说话人数），再学 5 人，最后学 8+ 人

## Part 4：三篇论文对 SpeakerMem-R1 设计的综合启示

### 4.1 Adaptive Credit Modeling 的 SpeakerMem 版本

借鉴 CoMAM 的 rank consistency 思想，设计 **speaker-level adaptive credit**：

```python
def compute_speaker_credit_weight(
    speaker_id: str,
    per_speaker_local_rewards: dict,   # {speaker_id: local_reward}
    global_reward: float,
) -> float:
    """
    对每个 speaker 的 memory operation，计算其 local reward 
    与 global QA reward 的 rank consistency → speaker-specific credit weight
    """
    local_reward = per_speaker_local_rewards[speaker_id]
    # 简化：直接用 local/global 相关性估计
    credit = 0.5 + 0.5 * sign(local_reward - mean(per_speaker_local_rewards.values()))
    return credit
```

### 4.2 Speaker-Aware Fact Extraction（最重要组件）

基于 Memory-R2 消融：fact extraction 是整个 pipeline 的最薄弱环节。在多方场景，这个问题更严重。

**SpeakerMem 的 Speaker-Aware Fact Extractor**：

```
输入：多方对话片段 [Alice: ..., Bob: ..., Carol: ...]
输出：
  - Alice 的 facts: ["Alice 在字节跳动做 NLP", "Alice 上周去了三亚"]
  - Bob 的 facts: ["Bob 喜欢咖啡", "Bob 和 Alice 有工作合作"]  
  - Carol 的 facts: ["Carol 是 AI 创业者"]
  - 跨说话人关系: ["Alice 和 Bob 互相认识且有合作"]
  - 隐私标注: {"Bob 的薪资": {"visible_to": ["Carol"]}}
```

这与 DeltaMem 的 local fidelity 结合：每个 speaker 的 fact 提取质量用 per-speaker Levenshtein F1 来评估。

### 4.3 Curriculum 修订（综合三篇论文经验）

```

Stage 1（SFT + 3-speaker, 8-session）：
```

## Part 6：综合结论与 SpeakerMem-R1 v2 设计修订

### 6.1 三大发现

1. **三篇竞品均无代码**：我们无法直接复现，但也意味着 reviewer 无法轻易质疑我们的实现细节
2. **Fact Extraction 是最薄弱环节**（Memory-R2 消融）：我们的 speaker-aware fact extraction 是最关键的组件，应该在 paper 中着重强调
3. **DeltaMem τ 敏感性存在**：per-speaker τ calibration 是有价值的小创新，可作为 Idea 7 的核心

### 6.2 对方法 spec 的修订

更新 SpeakerMem-R1-v2-方法spec.md 的关键点（已在方法 spec 中体现）：
- adaptive credit 实现参考 CoMAM 的 rank consistency（简单可实现）
- Levenshtein τ 建议范围 0.5~0.7，加 per-speaker 自适应版本
- curriculum 阶段参考 Memory-R2 的 session 数量递进（8→16→32）
- co-learning 用共享 backbone（Memory-R2 方式），不是独立模型

### 6.3 Idea 清单更新（V3 版）

| Idea | V2 评分 | V3 更新评分 | V3 变化原因 |
|------|---------|------------|------------|
| 1. SpeakerMem-R1 v2 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 不变，仍最强 |
| 2. Speaker-Aware Fact Extraction（新 highlight）| - | ⭐⭐⭐⭐⭐ | Memory-R2 消融揭示 extraction 是最薄弱环节，多方场景这个问题更严重，有独立 paper 价值 |
| 7. SpeakerLevenshtein | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 不变 |
| per-speaker τ calibration（新小创新） | - | ⭐⭐⭐ | 基于 DeltaMem 的 τ 敏感性，自适应校准有明确实现路径 |

**新 Idea：Speaker-Aware Fact Extraction as Core Contribution**

Memory-R2 消融显示：仅训练 Fact Extractor（无 Memory Manager）F1=28.30，去掉 Fact Extractor（仅 Memory Manager）F1=45.34——Fact Extractor 的缺失让 F1 直接崩塌，而 Memory Manager 的缺失只降 4 F1。**信息提取是整个系统的命脉**。

在多方场景，这个问题 n 倍加重：
- Dyadic: "这条信息归属给唯一的另一方" → 几乎无歧义
- Multi-party: "Alice 说的关于 Bob 的事情，是 Alice 的信息还是 Bob 的信息？" → 高歧义

→ **SpeakerMem-R1 的最大独立贡献可以定位为：第一个对多方场景的 speaker-attributed fact extraction 进行 RL 训练**，而不仅仅是 "把 dyadic RL memory 扩展到多方"。
