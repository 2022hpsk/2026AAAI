# 多角色/群聊对话记忆 + RL训练：面向AAAI的实用型深度调研报告

## 第一部分：执行摘要

**研究定位坐标。** 用户处在三条快速演进曲线的交汇点：(1) "RL统一记忆策略"曲线——从Memory-R1（arXiv 2508.19828，双Agent ADD/UPDATE/DELETE/NOOP + PPO/GRPO）到Mem-α（arXiv 2509.25911，core/episodic/semantic三组件 + QA奖励）再到AgeMem（arXiv 2601.01885，LTM/STM统一为六类工具动作 + 三阶段GRPO），核心范式已从"外挂启发式记忆"转向"记忆操作即可学习动作"；(2) "多方/群聊记忆评测"曲线——2026年密集出现GroupMemBench（arXiv 2605.14498）、EverMemBench（arXiv 2602.01313）、SocialMemBench（arXiv 2605.17789）三个专门针对多方场景的benchmark，且都报告现有系统"急剧崩溃"；(3) "多Agent共享/私有记忆"曲线——Collaborative Memory（arXiv 2505.18279）、G-Memory（arXiv 2506.07398）开始系统化处理跨用户/跨Agent记忆边界。

**关键结论：用户方向坐标极佳但窗口正在关闭。** 三条曲线在2026年初尚未被任何单一工作打通——即"端到端RL统一记忆策略"（曲线1）几乎全部在dyadic单用户QA上验证，而多方记忆（曲线2/3）目前全部是training-free的检索系统或纯benchmark，**没有人把RL统一记忆策略用到多角色/群聊/说话人感知的记忆管理上**。这正是用户的机会。但GroupMemBench/EverMemBench/SocialMemBench已占据"多方记忆benchmark"赛道，用户若想做新benchmark需高度差异化；做"多方RL记忆方法"则窗口仍开放。

**领域关键空白与机会。** (a) 现有RL记忆动作空间是"speaker-agnostic"的——没有按说话人归属、受众建模来组织记忆；(b) 多方场景的奖励设计空白——现有奖励都是单一QA正确性，没有针对"说话人归属正确""受众适配""知识更新"的过程奖励；(c) 共享vs私有记忆的可学习边界——Collaborative Memory用固定access-control策略，没人用RL学这个边界。

**推荐的3条AAAI路线（详见第四部分）：**
1. **SpeakerMem-R1**：把"说话人归属"做进RL记忆动作空间，在GroupMemBench/EverMemBench上做端到端RL，主打"speaker-grounded记忆操作"——最稳、最可行。
2. **GroupPolicy / 共享-私有记忆的RL边界学习**：统一策略学习"写到共享记忆还是私有记忆"，主打多Agent协作记忆——创新性最高、风险中等。
3. **多方记忆过程奖励 + benchmark双贡献**：构造多方RL记忆训练集 + 过程奖励模型，附带一个聚焦"RL训练"而非纯评测的数据贡献——双贡献抗审稿风险，工作量最大。

## 第二部分：精选20篇文献深度精读

### 第1组：方法奠基组（7篇）

**1. AgeMem — Agentic Memory: Learning Unified Long-Term and Short-Term Memory Management (arXiv 2601.01885, Yu et al., 2026年1月)**
- **核心方法**：把LTM与STM管理统一进Agent策略，记忆操作暴露为工具动作（ADD/UPDATE/DELETE/RETRIEVE/SUMMARY/FILTER六类），与文本生成共享同一动作空间。统一RL formulation：在时刻t，状态s_t包含对话上下文C_t（STM）、长期记忆库M_t、任务T；动作a_t从"reasoning token + 记忆操作"的混合动作空间中选。三阶段渐进式RL + step-wise GRPO解决记忆操作轨迹"碎片化、不连续"导致的优势估计难题。奖励三分量：Task Reward（0-1，LLM judge判最终答案）、Context Reward（STM管理质量）、Memory Reward（存储/检索质量），等权加和。
- **实验**：5个benchmark（ALFWorld, SciWorld, PDDL, BabyAI, HotpotQA），2个backbone（Qwen2.5-7B-Instruct, Qwen3-4B-Instruct），平均41.96%/54.31%，比强baseline高4.82-8.57个百分点；vs noRL高8.53/8.72个百分点；HotpotQA Memory Quality 0.533/0.605。
- **对用户的具体借鉴**：这是用户方法的**直接母体**。①混合动作空间 + step-wise GRPO框架可整体套用，用户只需把动作空间从speaker-agnostic扩展为speaker-aware（如`ADD(content, speaker_id, audience)`）；②三分量奖励结构可直接改造成多方版本——把Memory Reward拆为"speaker归属正确性"+"受众适配性"；③"碎片化轨迹的masked trajectory拼接"思路必须在论文Related Work和方法对比中引用。**这是必须对比的最强baseline。**
- **关系**：竞争（同范式）+ 可整合（作为backbone）。用户必须明确"AgeMem在dyadic上做，我在多方/speaker-grounded上做"的差异化。

**2. Memory-R1 — Enhancing LLM Agents to Manage and Utilize Memories via RL (arXiv 2508.19828, Yan et al., 2025年8月)**
- **核心方法**：两个专职Agent。Memory Manager学结构化操作{ADD, UPDATE, DELETE, NOOP}维护记忆库；Answer Agent用Memory Distillation策略从RAG召回的至多60条候选记忆中先筛选再推理。两者都用outcome-driven RL（PPO和GRPO）训练，仅用结果奖励（答案与gold的exact match），最小监督。
- **实验**：在LOCOMO上以LLaMA-3.1-8B-Instruct为backbone，Memory-R1-GRPO相对最强baseline Mem0实现 **F1相对提升48%、BLEU-1相对提升69%、LLM-as-a-Judge相对提升37%**；**仅用152条训练QA对**即可训练，并泛化到LoCoMo/MSC/LongMemEval三个benchmark及3B–14B多尺度backbone。
- **对用户的具体借鉴**：①"Manager + Answer"双Agent分工是最干净的baseline架构，用户可对比"单一统一策略 vs 双Agent分工"；②"60条候选→distillation"的两阶段检索可直接用于多方场景的"先按speaker过滤→再distill"；③**仅152条QA对即可训练**是用户做"低数据RL记忆"叙事的强支撑，也是做ablation"过程奖励是否必要"的对照组。
- **关系**：竞争+baseline。必须在LOCOMO上正面对比。

**3. Mem-α — Learning Memory Construction via RL (arXiv 2509.25911, Yu Wang et al., 2025年9月)**
- **核心方法**：把记忆构建建模为序列决策；记忆架构含core/episodic/semantic三组件，各配专用工具。奖励来自下游QA准确率，加两个超参：压缩奖励β、记忆内容奖励γ。
- **实验/关键数值**：仅用最长30k token的实例训练，却能泛化到 **超过400k token（13×训练长度）** 的序列，具体可达MemoryAgentBench Multi-Doc的474K token；内存footprint约为retrieval-only的一半；**最优压缩奖励β=0.05**；去掉语义有效性奖励（令γ=0）会降低记忆连贯性。
- **对用户的具体借鉴**：①多组件记忆架构（core/episodic/semantic）可直接映射为多方场景的"共享语义记忆 + 每说话人episodic记忆 + 群组core记忆"三层；②β/γ双奖励超参的消融设计（含β=0.05的最优点、γ=0的退化）是用户论文消融章节的现成模板；③"训练短（30k）、泛化长（400k+）"的实验叙事是AAAI审稿人喜欢的强卖点，用户应复刻"在小群聊训练→大群聊泛化"。
- **关系**：可整合（架构基底）+ 竞争。

**4. MemAgent — Reshaping Long-Context LLM with Multi-Conv RL-based Memory Agent (arXiv 2507.02259, ByteDance Seed & 清华AIR, 2025年7月)**
- **核心方法**：分段读文本，用overwrite策略更新固定长度token级记忆；扩展DAPO算法做independent-context multi-conversation生成。8K训练→3.5M token外推，性能损失<5%，512K RULER 95%+，线性复杂度O(N)。
- **对用户的具体借鉴**：①token级、人类可读的overwrite记忆使奖励建模和调试容易——用户若要处理超长群聊（数千turn）必须借鉴这种线性复杂度方案；②DAPO的multi-conversation扩展是处理"多说话人=多conversation流"的天然技术；③固定长度记忆对群聊"无界增长"问题是关键应对。
- **关系**：可整合（长上下文工程）。

**5. MEM1 — Learning to Synergize Memory and Reasoning for Efficient Long-Horizon Agents (arXiv 2506.15841, 2025年6月)**
- **核心方法**：推理与记忆consolidation统一在同一表征空间，不加额外模块。每turn持续更新上下文（合并旧记忆+裁剪无关token），near-constant内存。关键技术：masked trajectory approach——把多个演化上下文的turn拼接成逻辑连贯的完整轨迹，解决PPO/Reinforce++假设线性轨迹的问题。还提出task augmentation把单目标QA转成多目标任务。
- **对用户的具体借鉴**：①**masked trajectory是用户必用的技术细节**——多方记忆操作必然产生碎片化、上下文演化的轨迹，MEM1的拼接方法和AgeMem的step-wise GRPO是两个可选解；②"多目标任务增强"思路可用于把现有多方对话数据增强成多说话人多目标记忆任务。
- **关系**：可整合（训练技巧）。

**6. MemSearcher — Training LLMs to Reason, Search and Manage Memory via End-to-End RL (arXiv 2511.02805, Yuan et al., 2025年11月)**
- **核心方法**：每turn只给LLM"用户问题q + 紧凑memory M_{t-1}"，而非ReAct式拼接全历史；融合后生成推理轨迹、执行搜索、更新memory。提出multi-context GRPO：对同一query的一组轨迹传播标准化的trajectory-level优势，联合优化推理/搜索/记忆。3B超过7B baseline，多turn近似常数token。
- **对用户的具体借鉴**：①multi-context GRPO是处理"同一群聊问题、多条记忆管理轨迹"的优势分配方案，可直接迁移；②"固定大小memory + 当前turn"的输入格式是群聊上下文爆炸的解药；③3B超7B的token效率叙事可作卖点。
- **关系**：竞争+可整合。

**7. Mem0 + A-MEM + MemoryBank（记忆系统基线三件套）— 合并精读**
- **Mem0（arXiv 2504.19413, Chhikara et al.）**：production级，LLM驱动ADD/UPDATE/DELETE/NOOP，对每个fact用向量检索语义相似记忆再让LLM决定操作；图变体Mem0g用Neo4j建模实体关系、比base高约2%总分。关键数值：在LLM-as-a-Judge指标上**相对OpenAI记忆方案提升26%**，**p95延迟降低91%、token成本节省90%+**；评测用LOCOMO（10段长对话、每段约600 turn、约26,000 token、平均约200问）。
- **A-MEM（arXiv 2502.12110）**：Zettelkasten式原子记忆单元，动态链接、不需预定义schema，比MemoryBank灵活；检索时间3.70µs级。
- **MemoryBank（Zhong et al.）**：基于艾宾浩斯遗忘曲线的记忆强度动态更新 + 用户画像渐进构建。
- **对用户的具体借鉴**：这三个是**training-free baseline的标配三件套**，用户在LOCOMO/LongMemEval/多方benchmark上都必须列出它们作为"无RL"对照。Mem0的低延迟/低token成本是用户论文做效率对比时的参照数据。A-MEM的原子记忆+链接思想可用于"按speaker链接记忆"。MemoryBank的遗忘曲线可作为"何时DELETE"动作的先验。
- **关系**：baseline（必比）。

### 第2组：多角色场景组（6篇）

**8. G-Memory — Tracing Hierarchical Memory for Multi-Agent Systems (arXiv 2506.07398, Guibin Zhang et al., NUS/同济/UCLA/A\*STAR/NTU, 2025年6月)**
- **核心方法**：受组织记忆理论启发，用三层图层级管理MAS交互——insight graph（跨trial可泛化知识）、query graph（关联query/任务）、interaction graph（细粒度凝练的Agent间协作轨迹）。MAS表为有向图G=(V,E)，N个agent，T个同步通信epoch，按拓扑序π处理。检索：bi-directional memory traversal同时取高层insight和细粒度交互轨迹；更新：整个层级吸收新协作轨迹，无需改原框架。
- **实验**：5 benchmark（ALFWorld, SciWorld, PDDL, HotpotQA, FEVER）×3 backbone（GPT-4o-mini, Qwen2.5-7B/14B）×3 MAS框架（AutoGen, DyLAN, MacNet）。embodied成功率最高+20.89%（ALFWorld 58.21%→79.10%），QA准确率+10.12%。消融证明去掉insight或interaction graph都掉点；1-hop扩展通常最优，k∈{1,2}。
- **对用户的具体借鉴**：①三层图（insight/query/interaction）是多Agent协作记忆的**最佳结构基底**，用户的共享/私有记忆可对应到"interaction graph=私有协作轨迹、insight graph=共享提炼知识"；②bi-directional traversal检索可作为检索打分模块；③它是training-free的——用户用RL学这个层级的读写就是清晰的delta。
- **关系**：竞争（多Agent记忆）+可整合（结构）。**这是多Agent方向必比baseline。**

**9. Collaborative Memory — Multi-User Memory Sharing in LLM Agents with Dynamic Access Control (arXiv 2505.18279, Rezazadeh et al., Accenture, 2025年5月)**
- **核心方法**：多用户多Agent环境，非对称、时变access control编码为bipartite图（连接users/agents/resources）。两层记忆：private（仅创建用户可见）、shared（选择性共享）。每个fragment带不可变provenance属性（贡献agent、访问资源、时间戳）。write policy（π^write/private, π^write/shared）把交互投影为结构化fragment分配到两层；read policy按权限动态构造记忆view。全部组件用GPT-4o。
- **对用户的具体借鉴**：①**共享vs私有记忆的双层架构是用户Idea 2的直接起点**——但它的策略是固定/规则的，用户的delta就是"用RL学write policy该写private还是shared"；②provenance属性（谁说的、何时）正好是speaker-grounded记忆需要的元数据schema；③bipartite access图可作为多Agent权限边界的形式化工具。
- **关系**：竞争+可整合。**Idea 2必比baseline。**

**10. MuPaS — Multi-Party Supervised Fine-tuning for Multi-Party Dialogue Generation (arXiv 2412.05342, 2024年12月)**
- **核心方法**：多方对话微调框架，对每个role计算fine-tuning loss并mask掉非活跃role部分；两种训练策略（MuPaS-Speaker, MuPaS-Silence）把它转成MPD模拟器。SOTA多方response、更高next-speaker预测准确率，可泛化到OOD场景/话题/role描述。基于Qwen2-7B Instruct。
- **对用户的具体借鉴**：①**role-masked loss是处理"多说话人共享一个模型"的关键技术**——用户训练speaker-aware记忆策略时可用相同的mask机制；②next-speaker预测可作为一个辅助任务/辅助奖励来增强speaker建模；③MuPaS-Silence（决定何时不发言）的思想可类比"何时不写记忆"。
- **关系**：可整合（多方建模基底）。

**11. SHARE — Shared Memory-Aware Open-Domain Long-Term Dialogue Dataset from Movie Scripts (arXiv 2410.20682, Eunwon Kim et al., 2024年10月)**
- **核心方法**：从电影剧本构建的长期对话数据集，捕捉不同关系间的共享记忆。含两人persona信息、个人事件摘要，以及可隐式抽取的共享记忆。配套EPISODE框架：摘要persona+个人事件、抽取共享记忆并融入response生成。SHARE w/ predicted memory在perplexity、BLEU-3/4、ROUGE-1/2上更优。
- **对用户的具体借鉴**：①"共享记忆"在两人关系语境下的定义和抽取方法可扩展到群聊"群组共享记忆"；②电影剧本是天然的多角色长对话数据源，用户做数据时可借鉴这种构造管线。
- **关系**：可整合（数据构造）。**注意：见Caveats关于"SHARE"歧义说明。**

**12. MOOM — Maintenance, Organization and Optimization of Memory in Ultra-Long Role-Playing Dialogues (arXiv 2509.11860, Weishu Chen et al., 2025年9月)**
- **核心方法**：首个双分支记忆插件，借文学理论把"情节发展"和"人物刻画"作为核心叙事元素。分支1按多时间尺度总结情节冲突；分支2用预定义关键词抽取并更新用户角色profile。引入受"竞争-抑制"记忆理论启发的遗忘机制约束记忆容量、抑制无界增长。配套ZH-4O中文超长角色扮演数据集（平均600 turn，人工标注记忆）。outperform所有SOTA记忆抽取方法，更少LLM调用且容量可控。
- **对用户的具体借鉴**：①**"竞争-抑制"遗忘机制是DELETE/FORGET动作的理论依据**，用户可把它做成RL动作的奖励先验；②双分支（情节/人物）结构可类比"事件记忆/角色记忆"双轨；③600 turn超长对话场景说明角色扮演记忆的真实难度，用户应在此类数据上验证。
- **关系**：可整合（角色扮演记忆+遗忘机制）。

**13. SA-LLM — Contrastive Speaker-Aware Learning for Multi-party Dialogue Generation (arXiv 2503.08842, Tianyu Sun et al., 2025年3月)**
- **核心方法**：每条utterance显式标speaker标识符的speaker-aware输入编码；在标准LM loss外加对比学习目标，惩罚与speaker角色不一致的response，无需关系标注。在Ubuntu IRC和Movie Dialogues上超SOTA。
- **对用户的具体借鉴**：①**speaker-attributed输入编码是speaker-grounded记忆的最简实现**，可直接用在记忆条目的编码上；②对比学习目标可改造为"记忆-说话人一致性"的辅助损失，强化speaker归属。
- **关系**：可整合（speaker建模）。

### 第3组：评测基准组（5篇）

**14. GroupMemBench — Benchmarking LLM Agent Memory in Multi-Party Conversations (arXiv 2605.14498, Jingbo Yang et al., 2026年5月)**
- **核心方法/格式**：graph-grounded合成管线产出可控reply结构的多方对话，每条message基于per-user persona和target audience生成；adversarial query管线把每个问题绑定到特定asker，覆盖六类：multi-hop reasoning、knowledge update、term ambiguity、user-implicit reasoning、temporal reasoning、abstention。它显式测三个被忽视属性：群组动态、speaker-grounded belief tracking、audience-adapted language（ToM导致的角色专属词汇）。
- **关键结果**：最强系统仅46.0%平均准确率，knowledge update 27.1%，term ambiguity 37.7%；简单BM25 baseline匹配或超过多数agent记忆系统——说明现有记忆ingestion抹掉了群组记忆依赖的结构和词汇特征。
- **对用户的具体借鉴**：①**这是用户主实验的核心benchmark**——它的六类query正好定义了用户方法要优化的能力维度；②"BM25超过agent记忆系统"是用户论文动机段的强有力证据；③per-asker绑定、audience-conditioned的数据格式直接定义了用户speaker-aware记忆的输入输出schema。
- **关系**：评测平台（必用）。

**15. EverMemBench — Evaluating Long-Horizon Memory for Multi-Party Collaborative Dialogues (arXiv 2602.01313, 2026年2月)**
- **核心方法/格式**：多方多群组对话，跨1M+ token，密集cross-topic交织、时序演化决策、role-conditioned persona。三阶段管线：阶段2按项目计划+带日期公开新闻构造日历对齐时间线，增量合成长程群聊块；阶段3生成evidence-grounded QA并三阶段质控（LLM盲测、可答性/唯一性验证、人工审计）。2400 QA对，三维度：fine-grained recall、memory awareness、user-profile understanding。
- **对用户的具体借鉴**：①百万token级、多群组是用户"长程群聊"的极限压力测试；②memory awareness和profile understanding两个维度是GroupMemBench没有的补充，用户应两个benchmark都报；③日历对齐时间线的合成方法可借鉴用于数据增强。
- **关系**：评测平台（必用）。

**16. SocialMemBench — Are AI Memory Systems Ready for Social Group Settings? (arXiv 2605.17789, 2026年5月)**
- **核心方法/格式**：五种社交群组原型（密友、家庭、娱乐、兴趣社区、熟人网络）×三个群组规模档（4-30成员），430 personas、7355对话turn、1031 QA对、9类问题。提出五种失败模式：single-stream conflation、temporal-state overwrite、entity merging at scale、missing cross-persona knowledge、norm-individual conflation。全context Gemini 2.5 Flash仅0.721；四个开源记忆框架（Mem0, LangMem, Graphiti, Cognee）聚在0.12-0.18。
- **对用户的具体借鉴**：①五种失败模式是用户case study和error analysis的现成分类框架；②"entity merging at scale"（大规模实体合并）和"norm-individual conflation"（群组规范vs个体例外混淆）是用户方法可主打解决的具体问题；③群组规模分档（4-30人）可做scaling分析。
- **关系**：评测平台（必用）+ 失败模式分类。

**17. LOCOMO — Evaluating Very Long-Term Conversational Memory (Maharana et al., ACL 2024, arXiv 2402.17753)**
- **核心方法/格式**：50个高质量超长对话，每个平均300 turn、约9K token、至多35 session，基于persona+时序事件图生成，含图像。三任务：QA（五类：single-hop, multi-hop, temporal, open-domain, adversarial）、event summarization、multimodal dialog generation。主指标F1（partial-match），近期常用LLM-as-judge accuracy。
- **对用户的具体借鉴**：①这是dyadic长期记忆的"事实标准"，用户必须报以证明方法在单用户场景不退化；②五类QA分类是几乎所有记忆论文的通用切分；③Memory-R1/Mem0/AgeMem都在此报数，便于横向对比。
- **关系**：评测平台（必用，dyadic对照）。

**18. LongMemEval (arXiv 2410.10813) + PersonaMem-v2（arXiv 2512.06688）— 合并精读**
- **LongMemEval**：500条人工问题测五大记忆能力：information extraction、multi-session reasoning、temporal reasoning、knowledge updates、abstention。补足LOCOMO不测"助手提供信息的recall"和"更新信息推理"的缺口。
- **PersonaMem-v2**：1000个persona、20000+偏好、300+场景、128k上下文，偏好隐式揭示。前沿LLM隐式个性化仅37-48%准确率。还展示RFT（reinforcement fine-tuning）提升个性化，并提出growing single human-readable memory的agentic框架，超过GPT-5。
- **对用户的具体借鉴**：①LongMemEval的knowledge updates/abstention两类是用户多方"知识更新"奖励的对照；②PersonaMem-v2证明RFT对个性化有效，是用户"RL训练记忆"路线的强支撑证据，且其"单一可读记忆随用户增长"思路可扩展为"随每个说话人增长"。
- **关系**：评测平台 + RL有效性证据。

### 第4组：思想启发组（4篇）

**19. ReasoningBank — Scaling Agent Self-Evolving with Reasoning Memory (arXiv 2509.25140, Ouyang et al., Google, 2025年9月)**
- **核心方法**：把agent自身的成功+失败交互轨迹蒸馏成可复用的高层reasoning策略（而非raw trajectory或success-only workflow），检索指导未来决策。配memory-aware test-time scaling（MaTTS）：分配更多compute产生多样经验→对比信号→更高质量记忆。web+SWE benchmark上effectiveness相对提升最高+34.2%，交互步数-16%。
- **对用户的具体借鉴**：①"从失败中学"的记忆蒸馏可改造为多方场景"从归属错误中学"；②MaTTS的"经验scaling"是新scaling维度叙事；③strategy-level记忆（非raw log）是用户记忆条目设计的高层选择。
- **关系**：思想启发（记忆抽象层级）。

**20. 多方对话经典与RL训练技巧 — 合并精读**
- **SI-RNN（Addressee and Response Selection, AAAI 2018, arXiv 1709.04005）**：多方对话中说话人扮演sender/addressee/observer三角色且随turn变化；用role-sensitive方式更新speaker embedding，联合选addressee+response。**这是AAAI自家经典，引用它能贴合AAAI审稿口味**——证明speaker-grounded、addressee建模在AAAI有传统。
- **WebAgent-R1（arXiv 2505.16421, EMNLP 2025）+ RAGEN（arXiv 2504.20073）+ 多turn RL实践指南（arXiv 2510.01132）**：WebAgent-R1提出M-GRPO（GRPO扩到多turn）+动态上下文压缩+异步rollout；RAGEN揭示多turn RL的"Echo Trap"（坍缩到重复模板）；实践指南系统对比PPO/RLOO/GRPO/DAPO在多turn的适配问题。
- **对用户的具体借鉴**：①M-GRPO、动态上下文压缩、异步rollout是用户多turn RL训练的工程必备；②RAGEN的"Echo Trap"是用户必须在论文里讨论并用KL正则/多样性奖励应对的已知坑；③DeltaMem（arXiv 2604.01560）指出"用下游QA准确率做proxy reward导致严重稀疏奖励、难收敛"——这是用户必须正面回应的方法论质疑，用过程奖励或操作级监督来缓解。
- **关系**：思想启发+训练技巧+风险预警。

## 第四部分：3条具体的AAAI论文Idea

### Idea 1：**SpeakerMem-R1 — Speaker-Grounded统一记忆策略的端到端RL**

**核心创新点（一句话）**：首次把"说话人归属"作为一等公民嵌入RL记忆动作空间——记忆操作从`ADD(content)`升级为`ADD(content, speaker_id, audience)`，用step-wise GRPO端到端学习speaker-grounded记忆管理，在多方记忆benchmark上大幅超越speaker-agnostic的AgeMem/Memory-R1。AAAI审稿人会接收，因为它是"明确技术贡献+扎实多benchmark实验+清晰delta"的标准范式。

**方法架构**：
- **系统总览**：单一LLM策略π_θ，状态s_t=(群聊上下文C_t, speaker-indexed记忆库M_t={m_i: (content, speaker_id, audience, timestamp)}, 任务T)。
- **关键模块**：(1) speaker-attributed编码（借SA-LLM，每条记忆带speaker标识符）；(2) 混合动作空间（借AgeMem）：reasoning token + {ADD, UPDATE, DELETE, RETRIEVE, SUMMARY}×speaker维度；(3) 检索打分：score(q, m_i)=cos(e_q, e_{m_i})+λ·𝟙[speaker匹配asker关注]；(4) 学习型遗忘（借MOOM竞争-抑制）。
- **数学公式**：奖励 R = α·R_task + β·R_attr + γ·R_aud，其中R_attr=speaker归属正确率（过程奖励，针对GroupMemBench user-implicit类），R_aud=受众适配度。GRPO目标：J(θ)=E[Σ_t A_t·log π_θ(a_t|s_t)] − κ·KL(π_θ‖π_ref)（KL防RAGEN式Echo Trap）。
- **训练流程**：backbone Qwen2.5-7B/Qwen3-4B；三阶段渐进RL（借AgeMem）：先冷启动SFT学动作格式→LTM动作RL→STM+LTM联合RL；用masked trajectory（借MEM1）拼接碎片化记忆操作轨迹。

**实验设计**：
- 主实验：GroupMemBench（六类）、EverMemBench（三维度）、SocialMemBench（九类），dyadic对照报LOCOMO+LongMemEval；baseline：BM25、Mem0、A-MEM、MemoryBank（training-free）+ Memory-R1、Mem-α、AgeMem（RL，speaker-agnostic）。预期：六类query平均从46%提升到60%+，knowledge update和term ambiguity提升最大。
- 消融：去speaker维度、去R_attr、去R_aud、去学习型遗忘、去KL正则。
- 案例：展示SocialMemBench五种失败模式中至少三种被修复（single-stream conflation, entity merging, missing cross-persona）。

**预期贡献**：(1) 首个speaker-grounded RL记忆动作空间；(2) speaker归属+受众适配双过程奖励；(3) 多方记忆三benchmark新SOTA；(4) 揭示并修复三类群组记忆失败模式。

**风险与应对**：①审稿人质疑"只是AgeMem加speaker维度"→强调过程奖励设计+三个多方benchmark的系统验证+失败模式分析；②稀疏奖励难收敛（DeltaMem警告）→用过程奖励+冷启动SFT缓解；③合成benchmark的真实性→补LOCOMO真实对照+人工case分析。

**工作量**：4×A100约3-4周训练（7B+GRPO），数据准备2周（直接用现成benchmark），实验+消融3周，写作2周。总计约10-12周，**可在AAAI周期内完成**。

### Idea 2：**GroupPolicy — 共享/私有记忆边界的可学习RL策略**

**核心创新点**：把Collaborative Memory的固定access-control write policy替换为**RL学习的共享/私有路由策略**——统一策略在每次写入时学习"这条信息应进群组共享记忆还是说话人私有记忆"，用下游多Agent协作任务成功率+隐私泄露惩罚做奖励。AAAI会接收因为它把"记忆边界"从工程规则升级为可学习决策，且有明确隐私/协作权衡的理论动机。

**方法架构**：
- **系统总览**：基于G-Memory三层图（insight=共享提炼、interaction=私有协作轨迹），但读写由π_θ控制。
- **动作空间**：WRITE_SHARED(content)、WRITE_PRIVATE(content, agent_id)、PROMOTE（私有→共享）、READ_CROSS（跨agent读，受权限约束）。
- **奖励**：R = R_collab（协作任务成功）− η·R_leak（不该共享的私有信息被泄露的惩罚，用provenance检查）+ μ·R_efficiency（token效率）。
- **形式化**：用bipartite access图（借Collaborative Memory）做硬约束mask，π_θ只在合法动作上分布。multi-context GRPO（借MemSearcher）联合优化。

**实验设计**：
- 主实验：G-Memory的5 benchmark（ALFWorld, SciWorld, PDDL, HotpotQA, FEVER）×MAS框架（AutoGen, DyLAN）；新增隐私泄露测试集。baseline：G-Memory、Collaborative Memory（固定策略）、无共享记忆。
- 消融：去PROMOTE动作、去泄露惩罚、固定vs学习路由。
- 案例：展示学习到的"何时该共享"策略与人类直觉一致的轨迹。

**预期贡献**：(1) 首个可学习共享/私有记忆路由RL策略；(2) 协作-隐私联合奖励；(3) 多Agent benchmark上协作提升+泄露下降的Pareto改进。

**风险与应对**：①隐私泄露指标如何定义→构造带敏感标签的合成任务；②与G-Memory差异→强调"它training-free、我端到端RL学读写"；③多Agent训练不稳→借M-GRPO+异步rollout。

**工作量**：约12-14周（多Agent环境搭建+隐私评测集构造更费时），创新性最高。

### Idea 3：**GroupReward — 多方记忆过程奖励模型 + 训练数据双贡献**

**核心创新点**：针对DeltaMem揭示的"下游QA做proxy reward导致稀疏、难收敛"问题，构造首个**多方记忆操作级过程奖励模型（PRM）**——对每个记忆操作（按speaker归属、受众、知识更新正确性）打过程分，并配套发布一个面向"RL训练"（而非纯评测）的多方记忆训练集。方法+数据双贡献抗审稿风险。

**方法架构**：
- **PRM训练**：用LLM judge对操作打三类标签（归属对/错/中性，借RPGBench的align/contradict/neutral范式）→训练7B PRM（借CharacterBench的CharacterJudge思路）。
- **策略训练**：R_step = PRM(a_t, s_t)（密集过程奖励）+ R_outcome（最终QA），GRPO优化。
- **数据贡献**：借EverMemBench日历对齐时间线 + SHARE电影剧本管线，构造带操作级标注的多方记忆训练集。

**实验设计**：
- 主实验：GroupMemBench/EverMemBench，对比"纯outcome reward（Memory-R1式）vs +PRM过程奖励"，证明收敛更快、性能更高。
- 消融：PRM三类标签各自贡献、PRM大小（7B vs LLM judge）。
- 案例：展示稀疏奖励下outcome-only坍缩、PRM稳定收敛的训练曲线（呼应RAGEN Echo Trap）。

**预期贡献**：(1) 首个多方记忆操作级PRM；(2) 面向RL训练的多方记忆数据集（带操作标注）；(3) 证明过程奖励解决记忆RL稀疏奖励；(4) 开源PRM+数据。

**风险与应对**：①PRM标注成本→用LLM judge自动标+人工抽审；②数据集与现有benchmark重叠→强调"训练用+操作级标注"的独特性；③双贡献摊薄深度→主推方法，数据作为支撑。

**工作量**：约14周（数据+PRM训练+策略训练三阶段），工作量最大但抗拒稿性最强。

## Caveats（重要说明）

1. **部分arXiv ID为2026年新预印本**（AgeMem 2601.01885、GroupMemBench 2605.14498、EverMemBench 2602.01313、SocialMemBench 2605.17789、DeltaMem 2604.01560等），尚未经同行评审，benchmark数值可能随版本变动，引用时需注明预印本状态。
2. **多方记忆benchmark均为合成数据**，与真实群聊存在分布差距（SocialMemBench作者亦承认其多数探针为开放假设），用户应补真实数据case分析。
3. **"SHARE"存在歧义**：文献中literal命名的SHARE是对话数据集（arXiv 2410.20682），若用户原意指多Agent共享记忆，则应指Collaborative Memory（arXiv 2505.18279）或INMS（arXiv 2404.09982，Rutgers，异步共享会话记忆池）——建议用户确认。
4. **RL记忆的稀疏奖励问题尚未被根本解决**（DeltaMem明确指出proxy reward难收敛），所有RL-记忆Idea都需正面应对这一方法论风险；建议优先采用过程奖励/操作级监督。
5. 工作量估计假设用户有≥4×A100且能复用现成benchmark；若需从零构造数据，周期应相应延长。
6. 本报告精选20篇而非罗列全部——已主动剔除对AAAI写作无直接价值的纯survey与重复系统，并补充了用户书单外但高价值的新工作（G-Memory、Collaborative Memory、GroupMemBench、EverMemBench、SocialMemBench、SA-LLM、ReasoningBank、WebAgent-R1/RAGEN、DeltaMem、PersonaMem-v2等）。
