这三个是 **training-free 记忆系统的"基线三件套"**——任何做 RL 记忆的论文都必须把它们列为对照。它们代表了 LLM 时代记忆系统的三种主要工程范式：(1) Mem0 的"LLM 驱动 CRUD"；(2) A-MEM 的"原子链接"；(3) MemoryBank 的"遗忘曲线"。我把它们逐个剖开，最后讲对你研究的整体借鉴。

---

## Part 1: Mem0 / Mem0g —— production 标准的 CRUD 范式

### 1.1 核心方法

Mem0（arXiv 2504.19413，Chhikara et al.，Mem0.ai 公司出品）是当前最主流的开源记忆系统。它的核心 pipeline：

```
新对话 turn → 用 LLM 抽取 facts
        ↓
对每个 fact：
  1. 用向量检索找语义相似的已有记忆（top-k）
  2. 让 LLM 判断 action：ADD / UPDATE / DELETE / NOOP
  3. 执行 action 到记忆库
```

具体每个 action 的逻辑：

- **ADD**：新 fact 与现有记忆没冲突，直接新增。
- **UPDATE**：新 fact 与某条旧记忆有"refinement"关系——"用户喜欢咖啡" → "用户最爱拿铁"。
- **DELETE**：新 fact 与旧记忆直接矛盾——"用户在腾讯" + "用户跳槽去字节" → 旧记忆失效。
- **NOOP**：新 fact 已经包含在某条旧记忆中，不需操作。

注意这套 action 设计 **比 Memory-R1 的 4 个动作要早**——Memory-R1 是把 Mem0 的 action 体系作为 RL learning target 的对象。所以你的方法对比 Memory-R1 时，背后真正的 baseline action 设计是 Mem0。

### 1.2 Mem0g —— 图扩展

Mem0g 是 Mem0 的图变体，加了 Neo4j 实体关系图。具体差异：

- 抽取的 fact 不仅是文本，还包含 (entity, relation, entity) 三元组。
- 检索时除了向量相似，还看图上的 1-hop / 2-hop 邻居。
- LLM 决定 action 时可以参考图结构（"这两个 entity 之前已经通过某条 relation 连过"）。

**性能**：Mem0g 比 Mem0 平均高约 2 个百分点。代价是依赖 Neo4j infrastructure，部署复杂。

### 1.3 关键数字

- LLM-as-Judge 上**相对 OpenAI 内置记忆 +26%**。OpenAI 的 ChatGPT memory feature 是闭源黑盒，能比它高 26% 说明 Mem0 的 action-based 设计是实打实有效的。
- **p95 延迟降低 91%**：每次 turn 处理的延迟从几秒降到几百毫秒。
- **token 成本节省 90%+**：因为不需要把全部历史塞进 prompt，只塞检索到的相关记忆。

这几个数字是 **Mem0 production-grade 的基础**——它现在被很多商业 AI 助手用作 memory backbone。

### 1.4 评测协议

Mem0 用 LOCOMO 做主评测：
- 10 段长对话
- 每段约 600 turn
- 每段约 26000 token
- 平均 200 个问题/段

这是当前 dyadic 长期记忆的事实标准评测。**你的论文里报 LOCOMO 数字是 token, 必须的 baseline 对照**。

### 1.5 对你做多角色的借鉴

**借鉴 1**：Mem0 的 4 个 action（ADD/UPDATE/DELETE/NOOP）是 RL action space 的事实标准起点。你的 speaker-aware action space 应该是它的超集——多加 speaker_id / audience 参数。

**借鉴 2**：Mem0 的延迟数据（p95 几百毫秒）是你做效率对比的参照。你的 RL 方法即使比 Mem0 慢 2-3 倍（GRPO 训练后推理仍然慢），仍在可接受范围。

**借鉴 3**：Mem0g 的图扩展可以借鉴用于"speaker-entity 图"——每个 speaker 是一个 entity，他们之间的关系（同事/朋友/敌对）是 edge。这种结构对多角色场景天然贴合。

## Part 2: A-MEM —— Zettelkasten 式原子链接

### 2.1 核心方法

A-MEM（arXiv 2502.12110）的灵感来自 Zettelkasten 笔记法（Niklas Luhmann 的卡片盒笔记系统）。核心思想：

- 每条记忆是一个**原子卡片**——只记一件具体的事。
- 卡片之间用**动态生成的链接**关联，不依赖预定义 schema。
- 检索时通过链接遍历相关卡片。

具体的卡片 schema：

```python
class AtomicMemory:
    id: str
    content: str  # 原子内容，一句话
    tags: List[str]  # LLM 自动生成的标签
    links: List[Tuple[str, str]]  # [(target_id, link_type)]
    embedding: List[float]
    created_at: datetime
```

链接的类型不是预定义的——LLM 在创建链接时自己决定类型。常见的链接类型：

- `extends`: 新卡片是旧卡片的延伸
- `contradicts`: 新卡片和旧卡片矛盾
- `exemplifies`: 新卡片是旧卡片的具体实例
- `related_to`: 弱关联

### 2.2 性能与效率

A-MEM 的检索时间是 **3.70µs 级**——比 Mem0 的几十 ms 快得多。原因：

1. 原子卡片更小（一句话）→ embedding 更密集
2. 链接遍历是图操作，比向量检索快
3. 不需要 LLM 在检索时判断（链接已经在创建时决定）

性能方面，A-MEM 比 MemoryBank 灵活但比 Mem0 略弱（约低 1-2 个百分点）。

### 2.3 对你做多角色的借鉴

**借鉴 1：原子粒度**。多角色场景下记忆粒度选择很关键——一条记忆是一整段对话？一个 turn？一个 fact？A-MEM 的"原子 = 一句话"的粒度是最细的，便于按 speaker 归属（每个原子可以标 speaker_id）。

**借鉴 2：动态链接 = speaker 关系图**。把 link 的 type 扩展成 speaker 关系——"speaker A 引述 speaker B 说的话"、"speaker A 反驳 speaker B"。这种带语义的链接是多角色场景的天然结构。

**借鉴 3：检索效率**。3.70µs 的检索意味着你的 RL 训练时每个 rollout 的检索开销极小，可以做大规模 sampling。这是工程性优势。

## Part 3: MemoryBank —— 艾宾浩斯遗忘曲线

### 3.1 核心方法

MemoryBank（Zhong et al.）的核心创新是把心理学的**艾宾浩斯遗忘曲线**引入记忆系统：

```
记忆强度 S(t) = S_0 · e^(-t/τ)

其中 τ 是衰减常数，t 是从上次回忆/强化的时间。
每次回忆/强化都会重置 t，使记忆"重新强化"。
```

具体的实现：

- 每条记忆有一个强度分数 S，初始 1.0。
- 每过一个时间单位，S 按指数衰减。
- 每次记忆被检索（被用到），S 重置回 1.0（强化）。
- 当 S 低于阈值，记忆被丢弃（遗忘）。

这种设计天然产生 **"常用的留住、不用的丢"** 的效果，模拟人类记忆。

### 3.2 用户画像（progressive profiling）

MemoryBank 的另一个特色是"渐进构建用户画像"。具体：

- 维护一个 user profile，包含 traits / preferences / goals 等字段。
- 每次新对话，用 LLM 抽取信息更新 profile 的相应字段。
- 检索时除了具体记忆，还附带 profile 摘要。

这种 profile + episodic 双层结构和 Mem-α 的 core/episodic 分层是同构的，只是 MemoryBank 是 training-free 的固定规则。

### 3.3 对你做多角色的借鉴

**借鉴 1**：遗忘曲线可以作为 DELETE 动作的**先验奖励**——长期未被使用的记忆，DELETE 它应该有正奖励。这给你的 RL 方法提供了过程奖励的一个组件。

**借鉴 2**：每个 speaker 一份 profile（per-speaker progressive profiling）→ 你的多角色场景下，每个说话人维护一份独立 profile，检索时附带相关 speakers 的 profile 摘要。

**借鉴 3**：MemoryBank 是最弱的 baseline（论文里通常排在 Mem0/A-MEM 之下），但它的遗忘机制概念清晰，是论文中讨论"何时遗忘"必须 cite 的工作。

## Part 4: 三件套的对比总览

| 维度          | Mem0/Mem0g          | A-MEM            | MemoryBank          |
|---------------|---------------------|------------------|---------------------|
| 记忆粒度      | fact 级（句子/短语）| 原子级（一句话） | event 级 + profile  |

| 延迟          | p95 几百 ms         | 检索 3.70µs      | 中等                |

## Part 5: 对你研究的整体借鉴

### 5.1 它们都是你必须的 baseline

无论你做 Idea 1/2/3，都必须在 LOCOMO 上报 Mem0/A-MEM/MemoryBank 作为 training-free 对照。这是 AAAI 审稿人的硬性预期。

具体列表：

| Benchmark      | 必比 baseline                                      |
|----------------|----------------------------------------------------|
| LOCOMO         | Mem0, A-MEM, MemoryBank, Memory-R1, AgeMem        |
| GroupMemBench  | Mem0, BM25, Memory-R1（扩到多方）, AgeMem（扩到多方） |
| EverMemBench   | Mem0, A-MEM, AgeMem                               |
| SocialMemBench | Mem0, LangMem, Graphiti, Cognee                   |

### 5.2 它们暴露的具体 limitation 是你的 motivation

**对 Mem0**：
- speaker-agnostic（fact 没有 speaker 标签）→ 你的 motivation 1
- 规则化的 ADD/UPDATE/DELETE → 你的 motivation 2（用 RL 学）
- 在多方 benchmark 上"BM25 ≈ Mem0"（GroupMemBench 的发现）→ 你的 motivation 3（现有记忆系统忽略了多方结构）

**对 A-MEM**：
- 链接类型是 LLM 启发式定的 → 你的 motivation（link 应该 speaker-aware）

**对 MemoryBank**：
- 单一 profile（无多人）→ 你的 motivation（per-speaker profile）

### 5.3 它们的工程选择是你的参考

- 延迟 budget：Mem0 p95 几百 ms → 你的方法不能慢太多
- 存储 footprint：Mem0g 用 Neo4j 显得"重"，A-MEM 仅用 sqlite 显得"轻" → 你的方法应该走轻量路线
- 数据格式：(content, embedding, metadata) 的三元组是事实标准 → 你可以直接复用

### 5.4 一个 specific 的 critique angle

GroupMemBench 论文里有一个杀手级发现：**"BM25 匹配或超过多数 agent 记忆系统"**。这意味着 Mem0/A-MEM 这些 production-grade 系统在多方场景下**比 dumb 的关键词检索还不如**。

这个发现是你论文 Motivation 段的核武器。可以这样写：

> "Despite training-free memory systems achieving strong results on dyadic benchmarks (e.g., Mem0 reaches 0.4+ F1 on LOCOMO), recent multi-party evaluations reveal a striking degradation: on GroupMemBench, a simple BM25 baseline matches or exceeds Mem0, A-MEM, and other agent memory systems. This suggests that **current memory ingestion strategies erase the speaker- and audience-grounded structural cues that group memory critically depends on**. Our work addresses this gap by..."

### 5.5 Related Work 写法建议

> "Training-free memory systems for LLM agents can be grouped into three lineages: (i) the CRUD lineage exemplified by Mem0 [Chhikara et al., 2024], where an LLM directly chooses ADD/UPDATE/DELETE/NOOP per fact; (ii) the atomic-link lineage typified by A-MEM [..., 2025], where atomic memory units are dynamically interlinked without predefined schema; (iii) the forgetting-curve lineage represented by MemoryBank [Zhong et al.], which models memory decay via Ebbinghaus-style strength dynamics. All three are speaker-agnostic and have been shown to underperform even BM25 on multi-party benchmarks [GroupMemBench, 2026], motivating speaker-grounded learned memory operations..."

## Part 6: 一个反思——为什么 training-free 还有意义

虽然你做的是 RL 训练方法，但 training-free 三件套仍然有不可替代的价值：

1. **冷启动**：当你没有训练数据时，training-free 系统能立刻 deploy。
2. **泛化保底**：当你的 RL 模型遇到 out-of-distribution 输入时，training-free fallback 不会崩坏。
3. **可解释性**：规则化 action 比 RL 学出的隐式策略更容易解释/调试。

所以在你的论文 Discussion 段可以诚实地说："**我们的方法在 in-distribution 性能上显著超过 training-free 基线，但 training-free 系统在 cold-start 和 OOD 场景下仍有不可替代的实用价值，二者应被视为互补而非替代关系。**" 这种 honest 的定位会增加论文的说服力。
