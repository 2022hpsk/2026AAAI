## Part 1：V3 本轮新发现论文（未在 V2 覆盖）

### 1.1 [重要竞品] Mem-T（arXiv 2601.23014，2026/1月）

**全名**：Mem-T: Densifying Rewards for Long-Horizon Memory Agents

**来自**：北京大学 + 南洋理工大学

**核心创新**：MoT-GRPO = Memory Operation Tree + hindsight credit assignment

**背景**：单条线性轨迹中，memory operation 序列很长（几十步），outcome reward 稀疏且延迟。

**方案**：
```
Memory Operation Tree (MoT):
  - 每步从 K 个候选 memory 操作中做 tree expansion（非线性 beam search）
  - 树节点 = intermediate memory state
  - 用最终 QA outcome reward 反向传播 → 每个中间操作节点获得 dense process signal
  
Hindsight Credit Assignment:
  - 对已执行的 memory 操作，回溯问"这个操作对最终答案有多少贡献？"
  - 用 LLM 作为 post-hoc judge 评估每个 operation 的 counterfactual value
  
Hierarchical Memory Database:
  - 轻量级分层结构：Level-0 (raw event) → Level-1 (episode summary) → Level-2 (semantic abstract)
  - 动态 update + multi-turn retrieval
```

**效果**：
- vs A-Mem/Mem0 提升达 **+14.92%**
- vs GAM 减少 inference token **-24.45%**

**对我们的关键冲击**：
1. MoT-GRPO 也是"稠密化 sparse reward"的方案，但方向不同于 DeltaMem（state diff）
   - DeltaMem：state alignment（memory 结果和 GT 的距离）→ 结果端
   - Mem-T MoT：tree search + hindsight（单操作的 counterfactual）→ 过程端
   - 我们的 Speaker-aware Levenshtein：per-speaker 分桶 state diff → 归因端
2. **三者正交，可组合**：MoT（过程稠密化）+ DeltaMem（结果稠密化）+ SpeakerLev（speaker归因精确化）
3. Mem-T 的 tree 结构适合多方场景：**speaker-conditioned tree branching**——针对不同说话人选择不同 memory 操作分支，也是我们的新扩展点
4. **必须 cite Mem-T** 作为"稠密奖励"前置工作，强调 speaker-aware 扩展是新贡献

### 1.2 [重要参考] HCAPO（arXiv 2603.08754，2026/3月）

**全名**：Hindsight Credit Assignment for Long-Horizon LLM Agents

**核心创新**：HCAPO = Hindsight Credit Assignment Policy Optimization

**问题**：GRPO 的 step-level Q-value 估计不准确 + value baseline 不对齐（因为 intermediate state 随轨迹不同而不同）

**方案**：
- 利用 LLM 自身作为 post-hoc critic，在 hindsight（知道结果之后）对每步操作做推理
- 生成 "如果第 t 步做了不同操作，结果会如何不同？" 的反事实估计
- 用这个 counterfactual Q-value 替代原始 GRPO 的 group-relative advantage

**与我们的关系**：
- HCAPO 解决 dyadic GRPO 的 credit assignment 精度问题
- 我们的多方场景：每个 speaker 的 memory 操作对全局答案的贡献更难分离（14 人群聊里，某一说话人的信息被删除了，影响是什么？）
- **可借鉴**：把 HCAPO 的 hindsight LLM critic 扩展为"speaker-conditioned hindsight"——知道结果后，询问"删除 speaker A 的某条记忆会如何影响答案？"
- 这也是一个新的技术贡献点

### 1.3 [重要参考] HiPER（arXiv 2602.16165，2026/2月）

**全名**：HiPER: Hierarchical Reinforcement Learning with Explicit Credit Assignment for LLM Agents

**核心创新**：把 LLM 策略分为 high-level planner（出 subgoal）+ low-level executor（执行多步 action），用分层优势估计（HAE）分别给两层赋予信用

**效果**：ALFWorld 97.4%、WebShop 83.3%（Qwen2.5-7B）

**与我们的关系**：
- HiPER 的分层结构对应我们的两层 memory 策略：
  - High-level：`WRITE_TO(speaker_layer)` vs `WRITE_TO(group_layer)` 决策
  - Low-level：具体操作序列
- 可借鉴 HAE 思想来处理"写哪层"和"如何写"的两级 credit 分离

### 1.4 [重要新基准] SocialMemBench 数字深挖（arXiv 2605.17789）

**新补充的关键数字**（V2 只知道概况，现在有精确数字）：

- **数据规模**：430 personas, 348 sessions, 7,355 turns, **1,031 QA pairs**
- **网络类型**：5 archetypes（close friends, family, recreational, interest community, acquaintance）
- **成员数**：3 tiers（4–30 members）

- **现有 RL 记忆系统（Memory-R1/AgeMem）从未在这个 benchmark 上训练或评测**

- multi-hop attribution accuracy：仅约 **26%**（即使有 oracle 完整对话也只有26%）
- **我们只需超过 0.35 就是 SOTA**，而 RL 方法应该能大幅超过 training-free 基线

## Part 3：DeltaMem 工程细节补充（来自搜索结果摘要）

### 3.1 Tau 阈值选择

根据搜索结果的摘录信息：
- **tau 太小（如 0.1）**：wrong reward signals，把低相似度的 memory 配对也算进去
- **tau 太大（如 0.9）**：大量 valid match 被过滤，reward 退化
- **经验范围**：约 0.5 ~ 0.7 为合理区间（论文有 ablation）
- **我们的扩展**：对多方场景，per-speaker 的 tau 可能不同（个性化信息密度不同）

### 3.2 R_state 公式的完整理解

```
步骤 1：对 predicted_memory_delta 和 gt_memory_delta 各生成 embedding
步骤 2：计算 cosine similarity matrix（大小 |Δ_pred| × |Δ_gt|）
步骤 3：用最优传输（如 Wasserstein）做 one-to-one matching（Hungarian/Sinkhorn）
步骤 4：过滤 similarity < tau 的 pair
步骤 5：对 matched pair 计算 local lexical fidelity（predicted 是否包含 gt 的关键词）
步骤 6：合成 soft-precision = matched(pred) / total(pred)
         soft-recall = matched(gt) / total(gt)
         R_state = 2 * precision * recall / (precision + recall)

权重：R_total = 0.1·R_format + 0.1·R_retrieval + 0.8·R_state_transition
```

**对我们的扩展**：
```python
# Speaker-aware Levenshtein for SpeakerMem-R1 v3
def speaker_levenshtein_reward(pred_memory, gt_memory, speakers):
    # 1. 按 speaker 分桶
    per_speaker_f1 = []
    for s in speakers:
        pred_delta_s = get_speaker_delta(pred_memory, s)
        gt_delta_s = get_speaker_delta(gt_memory, s)
        f1_s = levenshtein_f1(pred_delta_s, gt_delta_s, tau=0.6)
        per_speaker_f1.append(f1_s)
    
    # 2. cross-speaker leakage penalty
    leakage = compute_cross_speaker_contamination(pred_memory, speakers)
    
    # 3. 合成
    R_state = mean(per_speaker_f1) - 0.3 * leakage
    return R_state
```

## Part 4：Memory-R2 LoGo-GRPO 工程细节补充

### 4.1 Progressive Curriculum

根据搜索结果确认：
- **Stage 1**：8 sessions → 训练 memory 基本操作
- **Stage 2**：16 sessions → 中等难度
- **Stage 3**：32 sessions → 长程依赖
- 这解决了"长程任务直接训 RL 不收敛"的问题

### 4.2 LoGo-GRPO 的 speaker-conditioned 扩展

我们的 v3 扩展思路（比 V2 更精确）：

```python
# Memory-R2 原始 LoGo
def logo_grpo_step(traj, lambda_=0.5):
    # Global: trajectory-level
    G_global = trajectory_reward(traj)
    
    # Local: 从 shared intermediate state 重新 rollout
    shared_state = get_intermediate_state(traj, t_split=len(traj)//2)
    local_rollouts = [new_rollout(shared_state) for _ in range(G_local)]
    G_local = group_relative(local_rollouts)
    
    return G_global + lambda_ * G_local

# 我们的 speaker-conditioned LoGo (v3)
def speaker_logo_grpo_step(traj, speakers, lambda_=0.5):
    G_global = trajectory_reward(traj)
    
    shared_state = get_intermediate_state(traj, t_split=len(traj)//2)
    
    # 关键扩展：按 speaker_set 分层 local rerollout
    # 同一 speaker 集合下的 rollout 才做 group-relative 比较
    # 避免"不同 speaker 组合"的轨迹互相比较（不公平）
    G_local_per_speaker = {}
    for speaker_subset in stratify_by_speaker_group(traj, speakers):
        local_rollouts = [
            new_rollout(shared_state, condition_on=speaker_subset) 
            for _ in range(G_local // len(speaker_subsets))
        ]
        G_local_per_speaker[speaker_subset] = group_relative(local_rollouts)
    
    G_local = mean(G_local_per_speaker.values())
    return G_global + lambda_ * G_local
```

### 4.3 computational overhead 预估

- Memory-R2 local rerollout：大约 2-3× 原始 GRPO 的 GPU 时间（根据 rollout 数量）
- 我们的 speaker-conditioned 扩展：如有 N 个 speaker，会进一步增加到 3-4×
- 缓解：
  1. 不需要对所有 intermediate state 做 rerollout，只选 "divergence point"（speaker 进入/离开群聊的时刻）
  2. 用 LoRA 而非 full fine-tuning（节省 80% 内存）
  3. G=4 而非 G=8（标准 GRPO 的一半）

#### 5.2.3 完整奖励函数 v3

```python
def compute_reward_v3(trajectory, ground_truth, speakers):
    """
    6 分量奖励 v3
    """
    # 1. Task reward: 最终 QA 正确性（token-level F1，不用 binary EM）
    R_task = token_f1(trajectory.final_answer, ground_truth.answer)
    
    # 2. Speaker attribution reward: 每条 memory 的说话人归属正确性
    R_attr = speaker_attribution_f1(
        trajectory.final_memory, 
        ground_truth.speaker_assignments
    )
    
    # 3. Audience adaptation reward: 输出是否对当前受众适配
    R_aud = audience_adaptation_score(
        trajectory.outputs, 
        ground_truth.audience_profiles
    )
    
    # 4. Speaker-aware Levenshtein state diff reward (NEW: v3 精化)
    R_state = speaker_levenshtein_reward(
        trajectory.final_memory, 
        ground_truth.memory_state, 
        speakers,
        tau=0.6  # DeltaMem 建议的合理区间中值
    )
    
    # 5. Compression reward: 不要存太多无用信息
    R_compr = compression_ratio_reward(trajectory.final_memory)
    
    # 6. Privacy/access control reward (NEW: v3)
    R_privacy = access_control_reward(
        trajectory.read_cross_ops,  
        ground_truth.access_matrix  # 谁有权读谁的记忆
    )
    
    # 权重（DeltaMem 经验：state reward 权重大；我们 attr 也很重要）
    total = (
        R_task   * 1.0   +   # 最终任务准确性（不可或缺）
        R_attr   * 0.5   +   # speaker 归属（我们的核心贡献）
        R_aud    * 0.3   +   # 受众适配（GroupMemBench 特有）
        R_state  * 0.8   +   # dense state diff（DeltaMem 风格，权重大）
        R_compr  * 0.1   +   # 压缩效率
        R_privacy * 0.2      # 隐私控制（新增 v3）
    )
    return total

# 注：R_task 和 R_state 权重可以通过消融确定最优组合
# 消融设计：
# - 去掉 R_attr → "no speaker attribution" 组
# - 去掉 R_state → "sparse reward only" 组（= Memory-R1 style）
# - 去掉 R_state 换 MoT-GRPO tree → "tree-based dense" 组（= Mem-T style）
# - 全部保留 → "SpeakerMem-R1 v3" 完整版
```

#### 5.2.4 训练 pipeline v3（精化版）

```python
def train_speakermem_r1_v3():
    """
    三阶段训练，每阶段有明确 checkpoint 目标
    """
    
    # ===== Stage 1: MuPaS-style Role-Masked SFT =====
    # 目标：让模型学会基础 memory 操作格式 + speaker-aware 输出
    # 数据：GroupMemBench train set + EverMemBench train set（mixed curriculum）
    # Loss：role-masked cross-entropy（只对 memory agent 角色的 token 计算 loss）
    # Checkpoint：验证集 memory format accuracy > 0.8
    
    sft_stage = SFTStage(
        data=mixed_curriculum(GroupMemBench.train, EverMemBench.train),
        loss=role_masked_cross_entropy,
        backbone="Qwen2.5-7B-Instruct",
        lora=True,  # LoRA r=16 减小计算开销
        epochs=3
    )
    
    # ===== Stage 2: Joint multi-agent RL with sparse warming =====
    # 目标：学 memory construction + retrieval 的协同（CoMAM 思路）
    # 算法：step-wise GRPO（AgeMem）
    # 奖励：R_task + R_attr（暂不用 R_state，先 warm start）
    # Curriculum：从 8 session 开始（Memory-R2）
    # G=4（Curriculum Study 建议，binary reward 下 G=4 gradient 非零）
    
    warm_rl_stage = WarmRLStage(
        algorithm=StepwiseGRPO(G=4),
        reward=lambda t, gt: compute_reward_v3(t, gt, sparse=True),  # 只有 R_task + R_attr
        curriculum=ProgressiveCurriculum(stages=[8, 16, 32]),  # Memory-R2
        joint=True,  # construction + retrieval joint（CoMAM）
        steps=1000
    )
    
    # ===== Stage 3: Full reward RL with LoGo + speaker-conditioned =====
    # 目标：全奖励 + 公平 credit assignment
    # 算法：speaker-conditioned LoGo-GRPO（v3 核心技术）
    # 奖励：全 6 分量（R_task + R_attr + R_aud + R_state + R_compr + R_privacy）
    # 评测：每 200 步在 GroupMemBench/SocialMemBench/EverMemBench dev 上验证
    
    full_rl_stage = FullRLStage(
        algorithm=SpeakerConditionedLoGoGRPO(
            G_global=8,
            G_local=4,  # 每个 speaker subset 做 4 个 local rollout
            lambda_logo=0.5  # Memory-R2 的 λ，暂定 0.5
        ),
        reward=compute_reward_v3,
        eval_every=200,
        eval_benchmarks=[GroupMemBench, SocialMemBench, EverMemBench],
        steps=3000
    )
    
    return sft_stage >> warm_rl_stage >> full_rl_stage
```

### 6.1 Abstract（英文草稿，约 250 words）

Title: SpeakerMem-R1: Speaker-Grounded Reinforcement Learning for 
       Multi-Party Conversational Memory Management

Abstract:
Real-world AI assistants increasingly operate in multi-party settings—
group chats, collaborative workspaces, and social networks—where 
conversational history is produced by multiple speakers with distinct 
roles, knowledge, and audiences. Existing memory-augmented LLM agents 
are designed and trained exclusively for dyadic (one-on-one) 
conversations, leaving a critical gap: no method learns to manage 
memory in multi-party settings via reinforcement learning.

We introduce SpeakerMem-R1, the first RL-trained memory agent for 
multi-party conversations. SpeakerMem-R1 treats memory operations—
Write, Update, Delete, Summarize, Promote, Suppress, and ReadCross—
as learnable actions conditioned on speaker identity and intended 
audience, organizing memory into a five-layer structure spanning 
per-speaker episodic/profile/core memories and group-level interaction/
insight memories. 

To address the challenge of sparse and unattributed rewards in 
multi-party settings, we propose three technical innovations:
(1) Speaker-aware Levenshtein State Reward (SLR), which computes 
per-speaker memory-state alignment via optimal transport matching, 
providing dense step-level feedback that correctly attributes memory 
quality to each speaker; 
(2) Speaker-Conditioned LoGo-GRPO, a local-global credit assignment 
algorithm that performs local rerollouts stratified by speaker subsets, 
ensuring fair group-relative comparisons; 
(3) Joint multi-agent training of the memory construction and retrieval 
agents with adaptive cross-agent credit weighting.

We train SpeakerMem-R1 on a mixed curriculum of three multi-party 
benchmarks—GroupMemBench, EverMemBench, and SocialMemBench—and evaluate 
against strong baselines including AgeMem, Memory-R2, and DeltaMem 
adapted to multi-party settings. SpeakerMem-R1 achieves X.X% on 
GroupMemBench, X.X% on SocialMemBench, and X.X% on EverMemBench, 
outperforming the best adapted dyadic baseline by X.X%. Ablation studies 
demonstrate the contribution of each technical component, particularly 
the speaker attribution reward and the speaker-conditioned credit 
assignment.

### 6.2 Introduction 关键段落（中文草稿）

**第 1 段（problem）**：
> 现代 AI 助手越来越多地部署在多方场景中——Slack 频道、微信群、团队协作工具、在线课堂——然而，所有已发表的 RL 训练记忆智能体都在"两人对话（dyadic）"假设下设计和验证。这一假设有两个根本性缺陷：
> (1) 它假设所有信息来自同一说话人，无法区分"谁说了什么"；
> (2) 它假设输出总是面向同一受众，无法对不同受众适配输出。
> 当 AI 助手在 10 人群聊中工作时，这两个假设立刻崩溃。GroupMemBench / SocialMemBench 的实验结果触目惊心：现有记忆系统在多方场景的准确率只有 0.12-0.18，相当于"几乎没有记忆"。

**第 2 段（prior work gap）**：
> 2025-2026 年已有多个工作尝试解决 RL 记忆智能体的稀疏奖励问题（Mem-T MoT-GRPO、DeltaMem Levenshtein、Memory-R2 LoGo-GRPO）和不公平 credit assignment 问题（HiPER、HCAPO），但所有这些工作都在 dyadic 设定下做实验，并且都没有考虑 speaker 归属这个多方场景的核心挑战。我们在 2026 年 6 月对 arxiv 的全面检索中没有发现任何"多方对话 + RL 训练记忆"的工作（见 Table 1）。

**第 3 段（contributions）**：
> 我们提出 SpeakerMem-R1，主要贡献包括：
> - 首个针对多方对话的 RL 训练记忆智能体，填补方法空白
> - Speaker-aware Levenshtein 奖励（SLR）：逐 speaker 分桶的稠密状态对齐奖励
> - Speaker-conditioned LoGo-GRPO：公平 credit assignment for multi-party rollout
> - Joint multi-agent 训练框架，统一 construction + retrieval + access-control
> - 在三个多方 benchmark 上取得 SOTA，相比最强 adapted 基线提升 X.X%
