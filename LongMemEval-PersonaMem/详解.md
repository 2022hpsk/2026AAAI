LongMemEval（arXiv 2410.10813）和 PersonaMem-v2（arXiv 2512.06688）是 **dyadic 长程记忆评测**的另两个重要 benchmark，与 LOCOMO 形成互补。它们各自补充 LOCOMO 不测的维度：

- **LongMemEval**：补**助手回复信息的 recall + 知识更新推理**
- **PersonaMem-v2**：补**隐式个性化 + 大规模 persona 维度**

我把它们合并精读，并讨论对你研究的具体借鉴。

---

# Part 1: LongMemEval

## 1.1 "500 条人工问题" —— 精挑细选

LongMemEval 的核心数据：

- **500 条人工标注问题**（不大但极精）
- 覆盖五大记忆能力维度
- 配套 long-form 对话历史

500 个 question 听起来不多，但每个都经过人工 vetting 保证质量。和 LOCOMO 的 10000 自动生成 questions 形成 contrast——LOCOMO 数量大但有 LLM 偏差，LongMemEval 数量小但质量高。

## 1.2 "五大记忆能力" —— 评测维度

LongMemEval 的五个维度：

### 维度 1: Information extraction

经典记忆：从对话中提取并保存信息。

例：

```
History: "User: I'm vegetarian and allergic to nuts."
Question (later): "What dietary restrictions does the user have?"
Expected: "Vegetarian, allergic to nuts"
```

测**基础记忆 fidelity**。

### 维度 2: Multi-session reasoning

跨 session 推理。

例：

```
Session 1: "User: I have a dog named Max"
Session 5: "User: Max chewed my shoes today"
Question: "Who is Max?"
Expected: "The user's dog"
```

测**跨 session 信息整合**。

### 维度 3: Temporal reasoning

时间推理。

例：

```
Session 1 (Jan): "User: I'm in college"
Session 8 (Dec): "User: I started my first job last month"
Question: "When did the user graduate?"
Expected: "Around November"  (推断: 12月开始第一份工作 → 11月毕业)
```

测**时间 reasoning + 缺失信息推断**。

### 维度 4: Knowledge updates

旧信息被新信息更新。

例：

```
Session 1: "User: I work at Google"
Session 5: "User: I quit Google and joined Meta"
Question: "Where does the user work?"
Expected: "Meta"  (not Google, even though Google appears more recently in retrieval)
```

测**知识 freshness handling**。

### 维度 5: Abstention

正确拒答不知道的信息。

例：

```
History: 没有提到 user 的生日
Question: "When is the user's birthday?"
Expected: "I don't know" / "Not mentioned in the conversation"
Wrong: 任何具体日期
```

测 **knowing-what-you-don't-know** 的元认知能力。

## 1.3 与 LOCOMO 的关键差异

LongMemEval 补足了 LOCOMO 两个缺口：

### 缺口 1：助手回复的 recall

LOCOMO 主要测"user 说过什么"。LongMemEval 增加"助手回复过什么"——即测试 AI 自己说过的内容的 recall。这是真实长对话场景需要的——user 经常 reference 之前 AI 给的建议。

### 缺口 2：知识更新推理

LOCOMO 的 multi-hop 主要是事实组合。LongMemEval 的 knowledge update **强调时序冲突解决**——同一信息在不同时间有不同答案，要选 latest。

这两个缺口对你做多角色研究都直接相关：

- 多角色场景里 "AI 之前对 user A 说过什么" 和 "user B 之后说过什么" 之间可能冲突
- 多角色里 knowledge update 频率更高（多 source）

## 1.4 实验结果（论文报）

| Method        | Info Ext | Multi-Sess | Temporal | Knowl Upd | Abstain | Avg  |
|---------------|----------|------------|----------|-----------|---------|------|
| Mem0          | 0.55     | 0.42       | 0.35     | 0.28      | 0.40    | 0.40 |
| A-MEM         | 0.58     | 0.45       | 0.37     | 0.30      | 0.42    | 0.42 |
| Memory-R1     | 0.65     | 0.50       | 0.42     | 0.35      | 0.48    | 0.48 |

关键 observation：

- **Knowledge update 最弱**（0.28-0.35）—— 和 GroupMemBench 一致
- **Abstention 也弱**（0.40-0.48）—— 模型 hallucinate 严重

这两个 weak 维度是你方法可重点 attack 的。

## 1.5 对你的借鉴

**对所有 Idea**：

- **Abstention 维度** → 你方法应该明确处理 "我不知道" 场景
- **Knowledge update 维度** → 你的 UPDATE/DELETE 设计的实证 motivation

**对 Idea 3（GroupReward）**：

- 五大维度直接对应 PRM 的五个 sub-head

# Part 2: PersonaMem-v2

## 2.1 "1000 个 persona、20000+ 偏好" —— 大规模个性化

PersonaMem-v2 是 PersonaMem 的升级版（2025 年 12 月发布）：

- **1000 个 persona**
- **20000+ 偏好**（每个 persona 平均 20 个 preferences）
- **300+ 场景**
- **128k 上下文**

这是**目前最大规模的 personalization benchmark**。

### 2.1.1 "偏好隐式揭示" —— 关键设计

PersonaMem-v2 最有意思的设计：**偏好不是 user 明说**，而是从行为/选择隐式推断。

例：

```
User: "I'd like to go out for dinner tonight."
AI: "Where would you like to go?"
User: "Let's try the new ramen place."

（隐式偏好：user 喜欢日料？喜欢面食？喜欢尝鲜？）
```

正确的记忆系统应该 store 隐式偏好，而不只是 surface 对话内容。

### 2.1.2 为什么 "隐式" 重要

真实场景里 user 很少明说 "我喜欢 X"——他通过 choices 表达。能从 choices 学 preferences 是 personalization 的关键能力。

## 2.2 "前沿 LLM 仅 37-48%" —— 又一惊人数字

PersonaMem-v2 报告 GPT-4/Claude/Gemini 在隐式个性化任务上 **只有 37-48% 准确率**。

这说明：

- 当前 LLM 不擅长从行为推 preference
- 隐式个性化是 open problem
- 你的方法在这里有发挥空间

## 2.3 "RFT 提升个性化" —— RL 有效的直接证据

PersonaMem-v2 论文的另一个 finding：

> **Reinforcement fine-tuning (RFT) significantly improves implicit personalization, achieving SOTA on the benchmark.**

这是非常 important 的 finding——**它直接证明 RL 对记忆/个性化有效**。

这是你做 RL 方法的**强有力支撑证据**：

> "Recent work [PersonaMem-v2, 2025] demonstrates that RFT substantially improves implicit personalization in long-context settings. Our work extends this RL-based approach to speaker-grounded memory management in multi-party scenarios."

## 2.4 "Growing single human-readable memory" —— Agentic 记忆框架

PersonaMem-v2 还提出一个 framework：**single human-readable memory that grows with the user**。

具体：

```python
memory = ""  # Start empty

for turn in dialogue:
    # Read current memory
    context = memory + turn
    
    # Generate response + updated memory
    response, new_memory = LLM.generate(
        prompt=f"Current memory:\n{memory}\nNew turn: {turn}\nGenerate response and updated memory."
    )
    
    memory = new_memory  # overwrite

# At inference, just feed memory + question
answer = LLM.generate(memory + question)
```

注意这和 MemAgent 的 overwrite memory 是同源思想——**纯文本可读 memory**，by overwrite 增长。

不同：MemAgent 是固定长度，PersonaMem-v2 是**逐渐增长**——memory 可以随 user 长度增长（in 128k context cap 内）。

### 2.4.1 对你的多角色应用

**直接扩展**：每个 speaker 一份 growing memory。

```python
memory_per_speaker = {speaker_id: "" for speaker_id in all_speakers}

for turn in group_chat:
    speaker = turn.speaker
    
    # Update speaker's individual memory
    new_memory = LLM.update(memory_per_speaker[speaker], turn)
    memory_per_speaker[speaker] = new_memory
    
    # Also update group memory
    group_memory = LLM.update(group_memory, turn, considering_all_speakers=True)
```

这种 **per-speaker growing memory** 是非常清晰的 architecture。

## 2.5 对你的借鉴

**对 Idea 1（SpeakerMem-R1）**：

- **隐式偏好抽取** → 你的方法应该建模 implicit preferences per speaker
- **growing memory framework** → 你的 architecture 灵感

**对 Idea 2（GroupPolicy）**：

- 1000 个 persona、20000+ 偏好 → 隐私 sensitive 数据的样本
- per-persona growing memory → 直接 model 你的 per-speaker private memory

**对所有 Idea**：

- **RFT 有效的 evidence** → 你 motivation 的 cite
- **128k context cap** → 你的 scaling 测试目标

# Part 3: 两个 benchmark 合并的总体借鉴

## 3.1 LongMemEval + PersonaMem-v2 对你的 evaluation 策略

| Benchmark      | 测什么                | 你的角色                        |
|----------------|----------------------|--------------------------------|
| LOCOMO         | dyadic 长期 QA       | dyadic compatibility（必报）   |
| LongMemEval    | knowledge update + abstention | dyadic 深度评测（必报） |
| PersonaMem-v2  | 隐式个性化            | dyadic personalization（建议报） |
| GroupMemBench  | 多方 6 类 query       | 主战场                          |
| EverMemBench   | 1M+ token 多群组      | 主战场                          |
| SocialMemBench | 5 种失败模式          | 主战场                          |

这种**多 benchmark 矩阵**是 AAAI 论文的标准 evidence 结构。

## 3.2 强叙事支撑

PersonaMem-v2 的 "RFT 有效" finding 是你方法的**强 motivation cite**：

> "Recent evidence [PersonaMem-v2, 2025] shows that reinforcement fine-tuning substantially outperforms supervised approaches on implicit personalization, motivating the RL-based design of our multi-party memory framework..."

## 3.3 一个统一的认知科学角度

LongMemEval 的 "abstention" 维度和 PersonaMem-v2 的 "implicit" 维度都涉及**元认知**：

- abstention = knowing what you don't know
- implicit personalization = knowing what user doesn't say but implies

你的论文可以用**元认知**作为一个统一 framework：

> "Multi-party memory requires three layers of metacognition: (1) source awareness (who said it), (2) audience awareness (who should know it), (3) boundary awareness (what is known vs unknown). Existing systems fail at all three, as evidenced by..."

这种 high-level framing 提升论文 intellectual depth。

# Part 5: 一个 strategic 的 angle —— RFT 有效的 leverage

PersonaMem-v2 的 "RFT 有效" finding 对你论文的策略价值：

1. **降低 reviewer 的 skepticism**：reviewer 不会怀疑 "RL 能不能 work" —— 因为 PersonaMem-v2 已经证明 work。
2. **缩短你 motivation 章节**：你不需要花太多篇幅论证 "为什么 RL 适合记忆"——直接 cite PersonaMem-v2。
3. **提供 quantitative benchmark**：PersonaMem-v2 报的 RFT 提升幅度可作为你方法目标的参考。

**实操**：把 PersonaMem-v2 的相关 finding 放在 Introduction 第 2-3 段，作为 "Why RL?" 的支撑 cite。

# Part 6: 一个 quality 的反思 —— 这两个 benchmark 都有 limitation

实事求是地说：

### LongMemEval 的 limitation

- **500 question 较小**，统计 power 有限
- **dyadic only**
- **英文**

### PersonaMem-v2 的 limitation

- **persona 是合成**——可能与真实人物的复杂性有差距
- **隐式偏好的 ground truth 由 LLM 标**——有偏见
- **128k context 是 hard cap**——超过 cap 怎么办？

**对你的意义**：你应该 cite 但不完全 rely on——主要看 GroupMemBench/EverMemBench/SocialMemBench 这些多方 benchmark。LongMemEval/PersonaMem-v2 是 supplement。
